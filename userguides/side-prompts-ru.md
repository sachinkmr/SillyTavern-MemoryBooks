# 🎡 Side Prompts

Side Prompts — это дополнительные запуски промптов STMB для обслуживания чата. Они могут анализировать, отслеживать, резюмировать, чистить или обновлять вспомогательные заметки, чтобы обычный ответ персонажа не делал всю эту работу сам.

Используйте их, когда чату нужен постоянный трекер, отчёт по отношениям, список сюжетных линий, журнал изобретений, статус-лист NPC, таймлайн или похожий вспомогательный документ. Персонаж может продолжать отыгрыш. Side Prompt берёт на себя бумажную работу. ❤️

## Что такое Side Prompts

Side Prompt — это именованный промпт, который запускается отдельно от обычного ответа персонажа.

Он может создавать или обновлять:

- трекеры сюжета
- трекеры отношений
- заметки по NPC или фракциям
- списки инвентаря/ресурсов
- таймлайны
- доски тайн/улик
- трекеры изобретений или проектов
- отчёты по непрерывности
- заметки для очистки
- вспомогательные записи в стиле lorebook

Side Prompts отличаются от обычных воспоминаний. Воспоминания обычно сохраняют последовательные резюме сцен. Side Prompts чаще поддерживают постоянный документ состояния, который обновляется или перезаписывается.

Они также **не обязаны** возвращать JSON. Обычный текст и Markdown подходят, если конкретный промпт или цель сохранения не требуют более строгого формата.

## Когда их использовать

Используйте Side Prompts для структурированной вспомогательной работы.

Хорошие варианты:

- **Сюжетные точки:** активные линии, закрытые линии, незавершённые хвосты
- **Отношения:** доверие, напряжение, влечение, границы, цели
- **NPC:** что каждый NPC знает, хочет, сделал недавно или должен сделать дальше
- **Таймлайн:** даты, путешествия, травмы, дедлайны, обратные отсчёты
- **Состояние мира:** изменённые локации, объекты, фракции, ресурсы
- **Тайны:** улики, подозреваемые, противоречия, вопросы без ответа
- **Проекты:** изобретения, исследования, блокеры, расползание объёма, следующие шаги
- **Непрерывность:** вероятные риски галлюцинаций или недостающий контекст

Плохие варианты:

- всё, что должно появиться внутри следующего ответа персонажа
- расплывчатые промпты вроде «сделай историю лучше»
- огромные аналитические промпты, которые каждый раз создают эссе
- дублирование резюме памяти без отдельной задачи

Side Prompts — не магия. Расплывчатый Side Prompt — это просто организованная расплывчатость.

## Быстрая настройка

Нужна пошаговая версия с кликами? Используйте [Scribe-инструкцию по включению Side Prompts](https://scribehow.com/viewer/How_to_Enable_Side_Prompts_in_Memory_Books__fif494uSSjCmxE2ZCmRGxQ).

Короткий путь: откройте **Extensions**, откройте **Memory Books**, нажмите **Side Prompts**, выберите нужный промпт, включите его, при необходимости включите **Run automatically after memory**, затем нажмите **Save** и **Close**.

## Как работают запуски

Обычный запуск Side Prompt проходит примерно так:

1. STMB выбирает сообщения для проверки.
2. Side Prompt подготавливается.
3. Нужные макросы заполняются.
4. Модель генерирует вывод Side Prompt.
5. STMB проверяет вывод.
6. Результат просматривается, сохраняется, обновляется или пропускается согласно настройкам Side Prompt.

Ручные Side Prompts, Side Prompts после памяти и строки Side Prompt Set должны ощущаться как части одной системы. Они используют одно и то же общее поведение для предпросмотра, пакетной обработки, проверки пустых ответов, сохранений, остановки и уведомлений.

## Ручные запуски

Используйте `/sideprompt`, чтобы вручную запустить один Side Prompt.

Базовая форма:

```txt
/sideprompt "Prompt Name"
```

С диапазоном сообщений:

```txt
/sideprompt "Prompt Name" 10-20
```

С runtime-макросом:

```txt
/sideprompt "Relationship Tracker" {{npc name}}="Alice" 10-20
```

Используйте кавычки вокруг имён промптов с пробелами.

Ручные запуски лучше всего подходят для разовых проверок, точечных обновлений и промптов, которым нужны пользовательские значения макросов.

## Автоматические запуски после памяти

Некоторые Side Prompts могут автоматически запускаться после создания памяти.

Это полезно, когда трекер должен оставаться актуальным по мере развития чата. Например, трекер отношений или трекер сюжета может обновляться после каждой памяти.

Есть два режима после памяти:

- **Use individually-enabled side prompts** — старое поведение; может запускаться любой Side Prompt, у которого включено **Run automatically after memory**.
- **Use a named Side Prompt Set** — вместо этого запускается выбранный набор.

Выбранный Side Prompt Set заменяет отдельно включённые Side Prompts после памяти. Он **не** добавляется к ним. Это предотвращает дублированные запуски из-за старых чекбоксов, о которых пользователь забыл.

## Side Prompt Sets

Side Prompt Sets объединяют несколько Side Prompts в один упорядоченный рабочий процесс.

Набор — это упорядоченный список запусков, а не просто папка. Один и тот же Side Prompt может встречаться больше одного раза с разными значениями макросов.

Пример набора:

1. Relationship Tracker с `{{npc name}} = Alice`
2. Relationship Tracker с `{{npc name}} = Bob`
3. Plot Points Tracker
4. Scene Cleanup Notes

Так один шаблон промпта может поддерживать отдельные записи для разных NPC, фракций, локаций или проектов.

### Управление наборами

Откройте **🎡 Trackers & Side Prompts**, чтобы создать, редактировать, дублировать, удалить или переупорядочить наборы.

Каждая строка может включать:

- Side Prompt
- необязательную метку строки
- сохранённые значения макросов
- элементы управления дублированием/удалением
- элементы управления перемещением вверх/вниз

Строки запускаются сверху вниз. Ставьте базовые трекеры первыми, а промпты для очистки/отчётов — позже.

### Ручной запуск набора

Запустить набор с сохранёнными значениями:

```txt
/sideprompt-set "Set Name"
```

С диапазоном:

```txt
/sideprompt-set "Set Name" 10-20
```

Запустить переиспользуемый набор со значениями макросов:

```txt
/sideprompt-macroset "Relationship Pass" {{npc_1}}="Alice" {{npc_2}}="Bob" 10-20
```

Используйте `/sideprompt-macroset`, когда в наборе есть переиспользуемые токены, которым всё ещё нужны значения.

### Отсутствующие наборы или строки

Side Prompt Sets намеренно строгие:

- Если набор не выбран, используется поведение отдельно включённых Side Prompts после памяти.
- Если набор выбран, отдельно включённые Side Prompts после памяти игнорируются.
- Если выбранный набор был удалён, ничего не запускается, и STMB предупреждает вас.
- Если строка указывает на удалённый промпт, эта строка пропускается, и STMB предупреждает вас.
- Если строке всё ещё нужно значение макроса, эта строка пропускается, и STMB предупреждает вас.

Тихий fallback был бы хуже. Если выбранный рабочий процесс сломался, вы должны это знать.

## Макросы

Side Prompts могут использовать обычные макросы SillyTavern, например `{{user}}` и `{{char}}`.

Они также могут использовать runtime-макросы — плейсхолдеры, которые заполняются при запуске Side Prompt.

Пример runtime-макроса:

```txt
{{npc name}}
```

Ручной запуск:

```txt
/sideprompt "Relationship Tracker" {{npc name}}="Alice"
```

Сохранённое значение в наборе:

```txt
{{npc name}} = Alice
```

Переиспользуемое значение уровня набора:

```txt
{{npc name}} = {{npc_1}}
```

Затем запуск:

```txt
/sideprompt-macroset "Relationship Pass" {{npc_1}}="Alice"
```

### Советы по макросам

Используйте скучные имена:

```txt
{{npc name}}
{{npc_1}}
{{faction}}
{{project_name}}
```

Избегайте имён вроде:

```txt
{{the guy we mean}}
{{stuff}}
{{important person}}
```

Пробелы удобны для чтения в UI. Подчёркивания обычно менее раздражают в slash-командах.

Side Prompt с пользовательскими runtime-макросами не стоит включать для индивидуальной автоматизации, если нужные значения нигде не сохранены, например внутри строки Side Prompt Set. Автоматические запуски не могут остановиться и спросить, кем должен быть `{{npc name}}`.

---

## Диапазоны сообщений

Side Prompts могут запускаться по конкретному диапазону сообщений.

```txt
/sideprompt "Plot Points" 50-80
```

Если вы указали диапазон, STMB использует этот диапазон.

Если вы не указали диапазон, STMB использует обычное поведение «с последнего Side Prompt» с существующей логикой лимитов и checkpoint.

Для регулярного отслеживания проще использовать поведение «с последнего». Для отладки или точечной очистки явные диапазоны понятнее.

Компиляция диапазона Side Prompt должна соблюдать те же предпочтения по скрытым сообщениям, что и память, включая глобальную настройку unhide-before-memory.

## Как писать хорошие Side Prompts

У хорошего Side Prompt есть задача. У плохого Side Prompt есть вайб.

Чётко укажите:

- что он должен просмотреть
- что он должен обновить
- что он должен игнорировать
- какой формат он должен вывести
- насколько длинным должен быть вывод
- должен ли он заменить, отредактировать или дописать

### Намеренно держите вывод коротким

Трекеры раздуваются, если не сказать им обратное.

Слабо:

```txt
Update the relationship tracker.
```

Лучше:

```txt
Update the relationship tracker. Preserve useful facts, remove resolved or obsolete details, and keep each entry to 1-3 concise bullets. Output only the updated tracker.
```

Полезные ограничения:

```txt
Do not append a new section unless there is genuinely new information. Merge updates into existing entries when possible.
```

```txt
Remove resolved threads. Do not preserve stale speculation just because it appeared in the old tracker.
```

```txt
Output only the updated report. No commentary, no explanation, no preface.
```

### Используйте стабильные заголовки

Стабильные заголовки делают повторные обновления чище.

Хорошо:

```md
# Relationship Tracker

## Current Status

## Recent Changes

## Open Tensions

## Next Likely Developments
```

Плохо:

```md
# Here is my extensive and emotionally intelligent breakdown of everything that might be happening
```

### Не просите всё сразу

Side Prompt, который просит каждую деталь, обычно и выдаст каждую деталь.

Выбирайте то, что важно. Трекеру сюжета обычно нужны незакрытый крючок, что изменилось, кто знает и что требует продолжения. Ему не нужно каждое выражение лица в сцене.

### Делайте использование макросов очевидным

Хорошие имена:

```txt
Relationship Tracker - {{npc name}}
NPC Status - {{npc name}}
Faction Tracker - {{faction}}
```

Менее полезные имена:

```txt
Tracker 3
Update thing
Misc relationship prompt
```

Пользователям не должно быть нужно открывать весь текст промпта, чтобы понять, почему он спрашивает значение.

## Примеры

### Plot Points Tracker

Используйте это, когда в чате есть несколько активных сюжетных линий.

```txt
Update the plot points tracker based on the selected messages. Keep only active or recently resolved threads. Group by storyline. Output only the updated tracker.
```

Рекомендуемая форма:

```md
# Plot Points

## Active Threads

1. **Missing artifact** — Current status and latest clue.
2. **Rival faction** — What they want and what changed.

## Recently Resolved

1. **Old misunderstanding** — Resolved when Alice told Bob the truth.

## Needs Follow-Up

1. Who has the key?
2. Why did the guard lie?
```

### Relationship Tracker с макросом

Промпту требуется:

```txt
{{npc name}}
```

Ручной запуск:

```txt
/sideprompt "Relationship Tracker" {{npc name}}="Alice" 10-40
```

Строки набора:

| Строка | Side Prompt | Сохранённый макрос |
|---|---|---|
| 1 | Relationship Tracker | `{{npc name}} = Alice` |
| 2 | Relationship Tracker | `{{npc name}} = Bob` |

Это позволяет не создавать отдельные определения промпта для каждого NPC.

### Трекер изобретений или проектов

Используйте это, когда пользователь постоянно изобретает, исследует, строит или со временем меняет что-то.

```txt
Update the project tracker. Track only meaningful changes in goal, progress, blockers, scope, dependencies, or story relevance. Keep entries concise and ordered by first introduction.
```

Обычно это чище, чем сохранять десять записей памяти, каждая из которых говорит, что проект существует.

### Переиспользуемый проход по составу

Создайте набор с runtime-токенами уровня набора:

```txt
{{npc_1}}
{{npc_2}}
```

Запустите его:

```txt
/sideprompt-macroset "Cast Pass" {{npc_1}}="Alice" {{npc_2}}="Bob"
```

Переиспользуйте его позже:

```txt
/sideprompt-macroset "Cast Pass" {{npc_1}}="Mira" {{npc_2}}="Jonas"
```

Тот же набор. Другой состав. 💡

## Устранение неполадок

### Мой Side Prompt не запустился после памяти.

Проверьте:

- Память действительно запускалась?
- Side Prompt включён для запусков после памяти?
- Чат использует **Use individually-enabled side prompts**?
- Чат вместо этого использует Side Prompt Set?
- Промпту нужно значение макроса, которое не было передано?
- Промпт был удалён, переименован или перемещён?

Если чат использует Side Prompt Set, отдельно включённые чекбоксы после памяти игнорируются для этого чата.

### Мой Side Prompt Set не запустился.

Проверьте:

- Набор выбран для этого чата?
- Набор всё ещё существует?
- Все строки указывают на существующие Side Prompts?
- Все обязательные макросы имеют сохранённые или переданные значения?

Автоматические запуски не могут спрашивать недостающие значения. Сохраните значения макросов в наборе или запустите его вручную через `/sideprompt-macroset`.

### Одна строка была пропущена.

Вероятные причины:

- связанный Side Prompt был удалён
- связанный Side Prompt был переименован
- в строке есть неразрешённые макросы
- модель вернула пустой или недопустимый ответ

STMB должен предупреждать, а не притворяться, что всё сработало.

### Вывод слишком длинный.

Добавьте жёсткие лимиты:

```txt
Keep the full output under 300 words.
```

```txt
Use no more than 5 active items.
```

```txt
Merge related details. Remove stale, resolved, or redundant details.
```

Модели сами по себе не знают, когда трекер стал бесполезно большим. Скажите им.

### Он запустился дважды.

Проверьте:

- ручной запуск плюс автоматический запуск
- дублирующиеся строки внутри набора
- повторные копии одного и того же Side Prompt
- несколько чатов или вкладок, запускающих работу почти одновременно

Выбранный Side Prompt Set должен заменять отдельно включённые промпты после памяти; это предотвращает одну распространённую причину дублированных запусков.

### Были проанализированы не те сообщения.

Используйте явный диапазон:

```txt
/sideprompt "Plot Points" 50-80
```

Поведение «с последнего» удобно. Явные диапазоны лучше для отладки.

### Трекер сохраняет устаревшую информацию.

Скажите Side Prompt удалять устаревшую информацию.

```txt
Update the tracker. Remove obsolete speculation, resolved conflicts, and details contradicted by the selected messages.
```

Трекеры не остаются чистыми сами по себе.

## Главное

### Для пользователей

Используйте Side Prompts, когда хотите структурированную помощь с поддержанием длинного чата.

Ручные запуски лучше всего подходят для разового анализа. Запуски после памяти или Side Prompt Sets лучше подходят для трекеров, которые должны оставаться актуальными.

### Для ботмейкеров

Создавайте Side Prompts как инструменты обслуживания, а не как ролевую прозу.

Используйте стабильные заголовки, строгие правила вывода и понятное поведение обновления. Используйте макросы, когда один промпт должен работать для нескольких NPC, фракций, локаций или проектов.

### Для админов

Side Prompts добавляют больше генеративной работы.

Это значит, что они должны быть предсказуемыми, проверяемыми и скучными в лучшем смысле. Наборы помогают, потому что делают предполагаемый рабочий процесс явным, а не оставляют его в супе из чекбоксов.
