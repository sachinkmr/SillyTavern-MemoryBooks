# 📕 ST Memory Books - AI 채팅 기억 보조 도구

**끝없는 채팅 대화를 체계적이고 검색 가능한 기억으로 바꿔보세요!**

봇이 내용을 기억하길 원하지만 문맥(Context)이 너무 길어졌나요? 줄거리를 잃지 않기 위해 수동으로 메모하는 것이 귀찮으신가요? ST Memory Books(STMB)는 바로 이런 문제를 해결해 줍니다. 이 확장 프로그램은 채팅을 지켜보다가 스마트한 요약을 생성하여 다시는 이야기의 흐름을 놓치지 않게 도와줍니다.

(기술적인 작동 원리가 궁금하신가요? [STMB 작동 원리(한국어)](howSTMBworks-ko.md) 문서를 참고하세요.)

## 🚀 빠른 시작 (첫 번째 기억 생성까지 5분!)

**ST Memory Books가 처음이신가요?** 몇 번의 클릭만으로 첫 번째 자동 기억을 설정해 봅시다.

### 1단계: 확장 프로그램 찾기

* 채팅 입력창 옆에 있는 마법 지팡이 아이콘(🪄)을 찾으세요.
* 아이콘을 클릭한 후 **"Memory Books"**를 선택하세요.
* ST Memory Books 제어 패널이 열립니다.

### 2단계: 자동 기능(Auto-Magic) 켜기

* 제어 패널에서 **"메모리 요약 자동 생성"** 항목을 찾으세요.
* 스위치를 **ON**으로 켭니다.
* **자동 요약 간격**을 **20-30 메시지**로 설정하세요 (처음 시작하기 좋은 값입니다).
* **자동 요약 버퍼**는 처음엔 `0-2` 정도로 두는 것이 무난합니다.
* 먼저 수동 메모리 1개를 만들어 채팅을 초기화하세요.
* 이제 끝입니다! 🎉

### 3단계: 평소처럼 채팅하기

* 평소대로 대화를 계속하세요.
* 20-30개의 새로운 메시지가 쌓이면, ST Memory Books가 자동으로 다음 작업을 수행합니다:
* 마지막 처리 위치 이후의 메시지를 사용
* AI에게 요약 작성 요청
* 기억 컬렉션(Memory collection)에 저장
* 완료 시 알림 표시



**축하합니다!** 이제 자동 기억 관리 시스템이 구축되었습니다. 더 이상 지난 챕터의 내용을 잊어버릴 걱정이 없습니다!

## 💡 ST Memory Books의 실제 기능

ST Memory Books를 채팅 대화를 위한 **개인 AI 사서**라고 생각해보세요:

### 🤖 **자동 요약 (Automatic Summaries)**

*"신경 쓰고 싶지 않아요, 그냥 알아서 해주세요"*

* 백그라운드에서 채팅을 감시합니다.
* 매 X 메시지마다 자동으로 기억을 생성합니다.
* 긴 롤플레이, 창작 글쓰기, 장편 스토리에 완벽합니다.

### ✋ **수동 기억 생성 (Manual Memory Creation)**

*"무엇을 저장할지 직접 정하고 싶어요"*

* 간단한 화살표 버튼(► ◄)으로 중요한 장면을 표시합니다.
* 특별한 순간에 대해 원할 때 즉시 기억을 생성합니다.
* 핵심 줄거리나 캐릭터의 변화를 포착하는 데 좋습니다.

### 📊 **사이드 프롬프트 & 스마트 트래커**

*"관계도, 플롯, 스탯 등을 추적하고 싶어요"*

* 기억 생성 시 함께 실행되는 재사용 가능한 프롬프트 조각들입니다.
* 바로 사용할 수 있는 템플릿 라이브러리를 제공합니다.
* 원하는 것은 무엇이든 추적하는 커스텀 AI 프롬프트를 만들 수 있습니다.
* 점수판, 관계 상태, 줄거리 요약 등을 자동으로 업데이트합니다.
* 예시: "누가 누구를 좋아하는가?", "현재 퀘스트 상태", "캐릭터 기분 추적기"

### 📚 **기억 컬렉션 (Memory Collections)**

*모든 기억이 살아 숨 쉬는 곳*

* 자동으로 정리되고 검색 가능합니다.
* SillyTavern에 내장된 로어북(Lorebook) 시스템과 연동됩니다.
* AI가 새로운 대화에서 과거의 기억을 참조할 수 있습니다.

## ✂️ 메모리 북에 클립

**메모리 북에 클립**은 전체 장면 기억을 만들지 않고 중요한 한 줄이나 사실만 저장하고 싶을 때 사용합니다. 채팅에서 텍스트를 선택하고, 플로팅 가위 버튼을 클릭한 다음 기존 클립 항목을 선택하거나 새 항목을 만드세요.

이것을 클립으로 저장해야 할지, 사이드 프롬프트로 관리해야 할지 헷갈린다면 [클립 vs 사이드 프롬프트](#️-클립-vs-사이드-프롬프트)를 참고하세요.

### 언제 클립을 사용해야 하나요?

클립은 AI가 기억해야 할 작은 사실에 적합합니다. 예:

- 캐릭터의 선호
- 약속이나 비밀
- 관계 세부 정보
- 반려동물, 장소, 아이템, 반복되는 세부사항
- 전체 기억 요약까지는 필요 없는 짧은 “나중에 기억할 메모”

더 큰 장면에는 일반 기억 생성을 사용하세요.

### 클립 작동 방식

1. 저장하고 싶은 문장이나 구절을 선택합니다.
2. 플로팅 가위 버튼을 클릭합니다.
3. 기존 클립 항목을 선택하거나 새 항목을 만듭니다.
4. 항목 미리보기를 검토합니다.
5. 클립을 저장합니다.

클립 항목은 `[STMB Clip]`으로 표시된 일반 로어북 항목입니다. 예:

```txt
Seraphina Healed Me [STMB Clip]
```

항목 내부에서 STMB는 내용을 깔끔한 섹션 형식으로 유지합니다.

```md
=== Seraphina Healed Me ===

- Seraphina healed my wounds with magic.

=== END Seraphina Healed Me ===
```

### 클립 항목 만들기 또는 이름 바꾸기

새 클립 항목을 만들면 항목 제목이 섹션 헤드라인도 됩니다. 클립을 저장하는 동안 항목 이름을 바꿀 수 있으며, STMB는 이에 맞춰 섹션 헤드라인을 업데이트합니다.

새 클립 항목은 다음 방식으로 만들 수 있습니다.

- **항상 활성**: 항상 사용할 수 있어야 하는 사실용
- **키워드 활성**: 일치하는 단어가 나올 때만 표시할 사실용

클립이 특정 주제, 캐릭터, 장소, 반려동물, 아이템, 관계에만 관련되어 있다면 키워드를 사용하세요.

### 플로팅 가위 버튼

플로팅 가위 버튼은 채팅 안의 텍스트를 선택한 뒤에만 나타납니다. 이 버튼은 Memory Books 메인 팝업에서 켜거나 끌 수 있습니다.

### 긴 클립 항목 검토

클립 항목이 길어지면 STMB가 검토를 요청할 수 있습니다. 직접 편집할 수도 있고, **압축**을 사용해 AI에게 클립, 사이드 프롬프트, STMB 메모리 항목을 원본과 교체하기 전에 더 토큰 효율적인 형태로 다시 쓰게 할 수도 있습니다.

## ✂️ 클립 vs 사이드 프롬프트

클립과 사이드 프롬프트는 둘 다 정보를 메모리 북에 저장하지만, 같은 용도의 기능은 아닙니다.

기본 규칙: **클립은 특정 사실을 저장합니다. 사이드 프롬프트는 살아 있는 트래커를 유지합니다.**

| **클립** | **사이드 프롬프트** |
|---|---|
| 선택한 채팅 텍스트를 메모리 북 항목에 저장합니다. | AI에게 채팅을 검토하게 하고 트래커 항목을 업데이트합니다. |
| 하나의 명확한 사실, 대사, 약속, 선호, 아이템, 메모에 적합합니다. | 관계 상태, 퀘스트 진행, 인벤토리, 미해결 플롯처럼 시간이 지나며 바뀌는 정보에 적합합니다. |
| 정확한 텍스트는 사용자가 선택합니다. STMB는 선택한 내용을 저장합니다. | AI가 채팅을 해석하고 트래커를 작성하거나 업데이트합니다. |
| 사실이 이미 분명하고 분석이 필요 없을 때 사용합니다. | AI가 여러 메시지를 비교, 요약, 상태 업데이트해야 할 때 사용합니다. |
| 보통 사용자가 다른 클립을 수동으로 추가할 때만 늘어납니다. | 이야기가 변하면 반복해서 업데이트될 수 있습니다. |
| 생각 방식: “이 메모를 고정한다.” | 생각 방식: “이 섹션을 계속 최신 상태로 유지한다.” |

좋은 클립 예:

- `Aiko는 꿀차를 좋아한다.`
- `Andalino는 다시는 그녀에게 거짓말하지 않겠다고 약속했다.`
- `Colt는 그녀를 Boss라고 부른다.`

좋은 사이드 프롬프트 예:

- 관계 상태
- 현재 퀘스트 진행
- 인벤토리와 자원
- NPC 디렉터리
- 미해결 플롯

기억할 세부사항이 하나뿐이면 클립을 사용하세요. 계속 갱신되는 트래커가 필요하면 사이드 프롬프트를 사용하세요.

## 🔎 주제별 클립

주제별 클립은 이미 만든 메모리에서 하나의 집중된 “이 주제에 대해” 항목을 만드는 기능입니다.

STMB에게 이렇게 요청한다고 생각하면 됩니다:

> “저장된 메모리를 읽고 이 인물, 장소, 관계, 줄거리, 아이템, 비밀 또는 주제에 대한 유용한 항목을 만들어 줘.”

이것은 여전히 클립 형식의 항목이지만, 선택한 채팅 텍스트를 클립하는 것은 아닙니다. 대신 STMB가 기존 메모리 항목을 원본으로 사용합니다.

간단한 규칙: **클립은 선택한 텍스트를 저장합니다. 주제별 클립은 저장된 메모리에서 관련 세부 정보를 모읍니다. 사이드 프롬프트는 시간이 지나며 트래커를 유지합니다.**

### 주제별 클립을 사용할 때

메모리 북에 이미 여러 메모리가 있고, 특정 주제에 대해 더 쉽게 활성화되는 항목을 만들고 싶을 때 사용합니다.

좋은 예:

- 반복 등장하는 NPC
- 두 캐릭터의 관계
- 미스터리나 조사
- 장소
- 세력
- 캐릭터의 능력, 부상, 약속, 비밀, 취향
- 여러 장면에 걸쳐 나타나는 줄거리

예시 주제:

```txt
Seraphina
{{user}}의 마법
Alex와 Mira의 관계
Black Harbor 조사
은색 열쇠
```

### 주제별 클립을 사용하지 않을 때

다음 경우에는 주제별 클립을 사용하지 마세요:

- 채팅에서 선택한 한 줄만 저장하고 싶을 때 — **메모리 북에 클립**을 사용하세요
- 이후 메모리 실행 중 자동으로 업데이트되는 트래커가 필요할 때 — **사이드 프롬프트**를 사용하세요
- 긴 항목 하나를 줄이고 싶을 때 — **컴팩션**을 사용하세요
- 여러 메모리를 상위 수준 요약으로 합치고 싶을 때 — **요약 통합**을 사용하세요

### 주제별 클립 사용 방법

1. Memory Books 팝업을 엽니다.
2. **🔎 주제별 클립**을 클릭합니다.
3. **원본 메모리 북**을 선택합니다.
4. **주제**를 입력합니다.
   - AI가 집중해야 할 대상입니다.
   - 구체적으로 적으세요.
5. **키워드**를 입력합니다.
   - 이는 lorebook 항목의 활성화 키워드가 됩니다.
   - 키워드를 비워 두면 STMB가 주제를 사용합니다.
6. 모드를 선택합니다:
   - **새 주제별 클립 만들기**는 새 `[STMB Clip]` 항목을 만듭니다.
   - **기존 항목 업데이트**는 기존 클립 항목을 업데이트합니다.
7. **생성 프로필**을 선택합니다.
   - 어떤 AI 연결/모델이 초안을 작성할지 정합니다.
8. 선택 사항: AI에게 보내는 지침을 바꾸고 싶다면 **주제별 클립 프롬프트 편집**을 클릭합니다.
9. **초안 생성**을 클릭합니다.
10. 생성된 초안을 검토합니다.
11. 필요하면 초안을 편집합니다.
12. **주제별 클립 저장**을 클릭합니다.

STMB는 초안을 자동으로 저장하지 않습니다. lorebook은 **주제별 클립 저장**을 클릭한 뒤에만 변경됩니다.

### 새 주제별 클립 만들기

새 주제별 클립을 만들면 STMB가 클립 형식의 lorebook 항목을 만듭니다.

예를 들어 주제가 다음과 같다면:

```txt
Seraphina
```

항목 제목은 다음과 비슷합니다:

```txt
Seraphina에 대해 [STMB Clip]
```

항목 안의 표시 섹션은 일반 클립 항목과 같은 클립 래퍼 스타일을 사용합니다.

### 기존 주제별 클립 업데이트

주제별 클립은 기존 `[STMB Clip]` 항목도 업데이트할 수 있습니다.

이미 다음과 같은 항목이 있을 때 유용합니다:

```txt
Seraphina에 대해 [STMB Clip]
```

그리고 마지막 업데이트 이후 새 메모리가 추가된 경우입니다.

주제별 클립 업데이트가 성공적으로 저장되면, STMB는 해당 항목에 작은 실행 기록을 저장합니다. 여기에는 실행 중 사용된 원본 메모리가 포함됩니다. 다음 업데이트에서 STMB는 이 기록을 사용해 모든 내용을 다시 읽는 대신 새롭거나 변경된 원본 메모리만 찾을 수 있습니다.

이렇게 하면 업데이트가 더 작아지고, 같은 오래된 메모리를 AI에게 반복해서 보내는 일을 줄일 수 있습니다.

### 모든 원본 메모리에서 다시 빌드

기존 주제별 클립을 업데이트할 때 **모든 원본 메모리에서 다시 빌드**가 보일 수 있습니다.

일반 업데이트에서는 꺼 둡니다. 가능하면 STMB는 새롭거나 변경된 원본 메모리만 사용합니다.

켜야 할 때:

- 기존 주제별 클립이 많이 오래되었을 때
- 주제별 클립 프롬프트를 변경했을 때
- 주제나 키워드를 크게 변경했을 때
- AI가 그 주제에 대한 모든 저장된 메모리를 다시 검토하게 하고 싶을 때
- 항목에 아직 유용한 실행 기록이 없을 때

### 어떤 원본 항목을 사용하나요?

주제별 클립은 선택한 메모리 북의 확인된 STMB 메모리 항목을 사용합니다.

사용하지 않는 것:

- 일반 클립 항목
- 사이드 프롬프트 트래커 항목
- STMB가 관리하지 않는 일반 lorebook 항목

이렇게 하면 주제별 클립이 STMB가 안전하게 식별할 수 있는 메모리에만 집중합니다.

### 주제별 클립을 잘 쓰는 습관

집중된 주제를 사용하세요.

더 좋음:

```txt
Alex와 Mira의 관계
```

덜 유용함:

```txt
이야기의 모든 것
```

더 좋음:

```txt
은색 열쇠
```

덜 유용함:

```txt
중요한 아이템
```

주제가 충분히 좁을수록 AI가 무엇이 포함되고 무엇이 제외되는지 판단하기 쉽습니다.

### 프롬프트 편집

주제별 클립 프롬프트는 편집할 수 있습니다.

기본 프롬프트는 AI에게 다음을 지시합니다:

- 주제와 관련된 정보만 추출하기
- 관련 없는 사건 피하기
- 이름, 관계, 취향, 약속, 비밀, 제약, 해결되지 않은 문제 보존하기
- 모순이 있으면 조용히 한쪽을 선택하지 말고 언급하기
- 기존 클립 내용을 중복 없이 업데이트하기
- 없는 세부 정보를 지어내지 않기

프롬프트에는 반드시 다음이 포함되어야 합니다:

```txt
{{SOURCE_MEMORIES}}
```

이 플레이스홀더가 없으면 STMB는 원본 메모리를 어디에 넣어야 할지 알 수 없습니다.

지원되는 다른 플레이스홀더:

```txt
{{MODE}}
{{TOPIC}}
{{KEYWORDS}}
{{EXISTING_CLIP}}
{{EXISTING_ENTRY_CONTENT}}
{{SOURCE_MEMORIES}}
```

사용자 지정 프롬프트가 잘 작동하지 않으면 **Reset to Default**를 사용하세요.

## 🙈 토큰 절약: 숨기기/다시 보이기

긴 채팅에서 정리를 쉽게 하고 토큰을 절약하는 가장 쉬운 방법 중 하나는, 이미 기억으로 만든 메시지를 숨기는 것입니다.

### “숨기기”는 무슨 뜻인가요?

숨기기는 **삭제가 아닙니다**. 메시지는 채팅에 그대로 남아 있고, 기억도 lorebook에 그대로 남습니다. 다만 AI에게 직접 보내지지 않을 뿐입니다.

### 언제 유용한가요?

* 채팅이 매우 길어졌을 때
* 해당 메시지로 이미 기억을 만들었을 때
* 채팅 화면을 더 깔끔하게 유지하고 싶을 때

### 기억 생성 후 자동 숨기기

STMB는 기억 생성 후 메시지를 자동으로 숨길 수 있습니다:

* **자동으로 숨기지 않음**: 자동으로 아무것도 숨기지 않음
* **마지막 메모리까지 모든 메시지 자동 숨기기**: 마지막 기억까지 포함된 모든 메시지를 숨김
* **마지막 메모리의 메시지만 자동 숨기기**: 가장 최근 처리 범위만 숨김

또한 **숨기지 않을 메시지 수**로 최신 메시지 몇 개를 계속 보이게 둘지 정할 수 있습니다.

### 기억 생성 전에 다시 보이기

**메모리 생성을 위해 숨겨진 메시지 숨김 해제 (/unhide X-Y 실행)**를 켜면 STMB가 기억 생성 전에 임시로 `/unhide X-Y`를 실행합니다.

### 추천 시작 설정

* **마지막 메모리의 메시지만 자동 숨기기**
* **2**개의 메시지를 보이게 유지
* **메모리 생성을 위해 숨겨진 메시지 숨김 해제 (/unhide X-Y 실행)** 켜기

## 🧭 압축 vs 통합

이름은 비슷하지만 서로 다른 일을 합니다.

기본 규칙: **압축은 항목 하나를 정리합니다. 통합은 여러 기억을 더 높은 단계의 요약 하나로 합칩니다.**

| **압축** | **통합** |
|---|---|
| STMB가 관리하는 기존 항목 하나를 더 작게 만듭니다. | 여러 기억이나 요약을 더 높은 단계의 요약 하나로 합칩니다. |
| 한 번에 Clip, Side Prompt 항목, 또는 STMB 메모리 항목 하나에만 작동합니다. | 선택한 여러 기억/요약 항목을 바탕으로 작동합니다. |
| 항목은 유용하지만 너무 길거나 반복적이거나 컨텍스트에 넣기 비쌀 때 적합합니다. | 오래된 장면 기억이 쌓여 Arc, Chapter, Book, Legend, Series, Epic 같은 요약으로 올려야 할 때 적합합니다. |
| 선택한 항목을 더 토큰 효율적인 형태로 다시 씁니다. | 선택한 원본 항목들로 새 요약 항목을 만듭니다. |
| 기존 사실을 보존하고 군더더기를 제거해야 합니다. | 더 큰 연속성 흐름을 보존하고 장면별 세부사항을 줄여야 합니다. |
| 생채팅에서 새 기억을 만들지 않습니다. | 부풀어진 항목 하나를 단독으로 압축하지 않습니다. |
| 생각 방식: “이 항목 하나를 다듬는다.” | 생각 방식: “이 기억들을 묶어 요약으로 올린다.” |

두 도구 모두 먼저 검토하는 방식입니다. STMB는 저장하거나 교체하기 전에 AI가 작성한 내용을 보여줍니다.

## 🌈 요약 통합

요약 통합은 오래된 STMB 기억을 더 높은 단계의 recap 항목으로 압축해 긴 이야기를 관리하기 쉽게 만드는 기능입니다.

### Q: 요약 통합이란 무엇인가요?

**A:** STMB는 장면 단위 기억만 계속 만드는 대신, 기존 기억이나 요약을 더 압축된 recap으로 합칠 수 있습니다. 첫 번째 단계는 **Arc**이고, 더 긴 이야기에는 더 높은 단계도 사용할 수 있습니다:

* Arc
* Chapter
* Book
* Legend
* Series

### Q: 왜 사용하나요?

**A:** 다음과 같은 경우에 유용합니다:

* 기억 목록이 길어졌을 때
* 오래된 항목에 더 이상 장면 단위 세부 정보가 필요 없을 때
* 연속성을 잃지 않고 토큰 사용량을 줄이고 싶을 때
* 더 깔끔하고 상위 수준의 이야기 recap을 원할 때

### Q: 자동으로 실행되나요?

**A:** 아니요. 통합은 여전히 확인이 필요합니다.

* 메인 팝업에서 언제든 **기억 통합하기**를 수동으로 열 수 있습니다
* **티어가 준비되면 통합을 묻기**도 켤 수 있습니다
* 선택한 대상 tier가 최소 충족 수에 도달하면 STMB가 **yes/later** 확인을 표시합니다
* **Yes**를 누르면 해당 tier가 선택된 상태로 통합 팝업만 열리며, 자동으로 조용히 실행되지는 않습니다

### Q: 어떻게 사용하나요?

**A:** 요약 통합을 만들려면:

1. STMB 메인 팝업에서 **기억 통합하기**를 클릭합니다
2. 대상 요약 tier를 선택합니다
3. 포함할 원본 항목을 고릅니다
4. 새 요약 생성 후 원본 항목을 비활성화할지 선택합니다
5. **Run**을 클릭합니다

AI가 통합 응답을 잘못 반환하면, 다시 저장하기 전에 검토/수정 흐름을 사용할 수 있습니다.

### 무엇이 통합되고, 무엇은 통합되지 않나요?

통합은 **STMB 기억과 STMB 요약**을 대상으로 합니다.

즉:

* 일반 기억은 더 높은 단계의 요약으로 통합될 수 있습니다
* 상위 요약도 더 큰 recap으로 다시 통합될 수 있습니다

사이드 프롬프트는 다릅니다.

**사이드 프롬프트는 트래커 항목**이지, 기억 요약 항목이 아닙니다. 사이드 프롬프트는 다음과 같은 지속적인 상태 정보를 최신으로 유지하기 위한 것입니다:

* 관계 상태
* 현재 목표
* 점수판
* 세계 상태
* 플롯 트래커

간단히 말하면:

* **기억** = "이 장면에서 무슨 일이 있었나?"
* **사이드 프롬프트** = "이것의 현재 상태는 무엇인가?"
* **통합** = "많은 기억을 더 큰 recap으로 요약하기"

## 🎨 트래커, 사이드 프롬프트 & 템플릿 (고급 기능)

**사이드 프롬프트**는 진행 중인 스토리 정보를 유지하는 데 도움을 주는 백그라운드 추적기입니다.
이들은 별도의 Side Prompt 항목을 lorebook에 만들고 기억 생성과 함께 실행됩니다. **스토리를 지켜보며 특정 세부 사항을 최신 상태로 유지해 주는 도우미**라고 생각하세요.
표준 ST 매크로인 `{{user}}`와 `{{char}}`는 `Prompt`와 `Response Format`에서 확장됩니다. 표준이 아닌 `{{...}}` 매크로는 수동 실행 시 필수 입력이 됩니다.

### 🚀 **템플릿으로 빠른 시작**

1. Memory Books 설정을 엽니다.
2. **사이드 프롬프트**를 클릭합니다.
3. **템플릿 라이브러리**를 둘러보고 스토리에 맞는 것을 선택하세요:
* **Character Development Tracker** – 성격 변화와 성장 추적
* **Relationship Dynamics** – 캐릭터 간의 관계 추적
* **Plot Thread Tracker** – 진행 중인 줄거리 추적
* **Mood & Atmosphere** – 감정적 톤 추적
* **World Building Notes** – 배경 설정 및 로어(Lore) 추적


4. 원하는 템플릿을 활성화합니다 (나중에 수정 가능).
5. 템플릿에 사용자 지정 런타임 매크로가 포함되어 있으면 자동으로 실행되지 않으며 `/sideprompt`로 수동 실행해야 합니다.

### ⚙️ **사이드 프롬프트 작동 원리**

* **백그라운드 트래커**: 조용히 실행되며 시간이 지남에 따라 정보를 업데이트합니다.
* **비침해적**: 메인 AI 설정이나 캐릭터 프롬프트를 변경하지 않습니다.
* **채팅별 제어**: 채팅마다 다른 트래커를 사용할 수 있습니다.
* **템플릿 기반**: 내장 템플릿을 사용하거나 직접 만들 수 있습니다.
* **자동 또는 수동**: 표준 템플릿은 자동으로 실행할 수 있고, 사용자 지정 런타임 매크로가 있는 템플릿은 수동 전용입니다.
* **매크로 지원**: `Prompt`와 `Response Format`은 `{{user}}`와 `{{char}}` 같은 표준 ST 매크로를 확장합니다.
* **안전 확인**: 템플릿에 사용자 지정 런타임 매크로가 있으면 STMB가 저장/가져오기 시 `onInterval`과 `onAfterMemory`를 제거하고 경고를 표시합니다.

### 🛠️ **사이드 프롬프트 관리**

* **사이드 프롬프트 관리자**: 트래커 생성, 편집, 복제 및 정리.
* **Enable / Disable**: 언제든지 트래커를 켜거나 끌 수 있습니다.
* **Import / Export**: 템플릿을 공유하거나 백업합니다.
* **Status View**: 현재 채팅에서 어떤 트래커가 활성화되어 있는지 확인합니다.

### 💡 **템플릿 예시**

* 사이드 프롬프트 템플릿 라이브러리 (JSON 가져오기):
[SidePromptTemplateLibrary.json](../resources/SidePromptTemplateLibrary.json)

프롬프트 아이디어 예시:

* "중요한 대화와 캐릭터 상호작용 추적"
* "현재 퀘스트 상태 최신화"
* "새로운 세계관 설정(World-building) 세부 사항이 나오면 기록"
* "캐릭터 A와 캐릭터 B 사이의 관계 추적"

### 🔧 **커스텀 사이드 프롬프트 만들기**

1. 사이드 프롬프트 관리자를 엽니다.
2. **새로 만들기**를 클릭합니다.
3. 짧고 명확한 지시사항을 작성하세요.
*(예: "각 장면의 날씨가 어떤지 항상 기록해 줘")*
4. 필요하면 표준 ST 매크로나 런타임 매크로를 `{{macro}}="value"` 형식으로 추가합니다.
5. 저장하고 활성화(Enable)합니다.
6. 자동 트리거가 활성화되어 있을 때만 이 정보가 시간이 지남에 따라 업데이트됩니다.

### 💬 **프로 팁**

사이드 프롬프트는 **작고 집중적**일 때 가장 잘 작동합니다.
"모든 것을 추적해" 대신 "주인공들 사이의 로맨틱한 긴장감을 추적해"와 같이 구체적으로 설정해 보세요.

---

### 🧠 Regex 확장 프로그램을 이용한 고급 텍스트 제어

ST Memory Books는 생성 전과 저장 전에 선택한 Regex 스크립트를 실행할 수 있습니다.

* STMB에서 **정규식 사용(고급)**를 켭니다
* **📐 정규식 구성…**를 클릭합니다
* AI에 보내기 전과 저장 전에 각각 어떤 스크립트를 실행할지 따로 선택합니다
* STMB 안에서 선택한 스크립트는 Regex 확장 프로그램에서 비활성화되어 있어도 실행될 수 있습니다

## 🧹 압축

압축은 STMB가 관리하는 로어북 항목이 여전히 유용하지만 너무 길거나 반복적일 때 사용하는 정리 기능입니다. 직접 잘라내는 대신 AI에게 항목을 더 토큰 효율적인 형태로 다시 쓰게 할 수 있습니다.

이 기능은 **먼저 검토하는 도구**입니다. STMB는 아무것도 교체하기 전에 원본과 압축 초안을 보여줍니다.

### 무엇을 압축할 수 있나요?

압축은 선택한 메모리 북에서 다음 항목을 표시할 수 있습니다:

- 클립 항목
- 사이드 프롬프트 트래커 항목
- STMB 메모리 항목

STMB가 관리하지 않는 일반 로어북 항목은 표시하지 않습니다.

### 압축 사용 방법

1. Memory Books 팝업을 엽니다.
2. **📝 압축**을 클릭합니다.
3. 검토할 **메모리 북**을 선택합니다. 현재 채팅에 메모리 북이 이미 있으면 자동으로 선택될 수 있습니다.
4. **압축 프로필**을 선택합니다. 이 프로필은 어떤 AI 연결/모델이 항목을 다시 쓸지 정합니다.
5. 필요하면 **압축 프롬프트 편집**을 클릭해 다시 쓰기 지시문을 수정합니다.
6. 표에서 항목을 찾아 **항목 압축**을 클릭합니다.
7. 결과를 검토합니다:
   - **원본 내용**은 현재 저장된 내용입니다.
   - **압축 초안**은 AI가 다시 쓴 내용입니다.
   - 둘 다 예상 토큰 수를 표시합니다.
8. 필요하면 압축 초안을 직접 편집합니다.
9. 다음 중 하나를 선택합니다:
   - **압축된 버전으로 바꾸기**: 초안을 원본 항목 위에 저장합니다.
   - **압축 초안 복사**: 저장하지 않고 초안만 복사합니다.
   - **취소**: 항목을 변경하지 않습니다.

STMB는 원본을 조용히 교체해서는 안 됩니다. **압축된 버전으로 바꾸기**를 클릭하지 않으면 로어북 항목은 그대로 유지됩니다.

### 압축 프롬프트 편집

압축 프롬프트는 AI가 항목을 어떻게 다시 쓸지 제어합니다. 내장 프롬프트는 의도적으로 보수적입니다. 중요한 사실, 이름, 대명사, 매크로, 래퍼 헤딩, 종료 마커는 보존하고, 반복과 낮은 가치의 표현은 제거하며, 없는 내용을 만들어내지 않도록 지시합니다.

프롬프트는 다음 플레이스홀더를 지원합니다:

- `{{ENTRY_CONTENT}}` — 현재 항목 내용입니다. 필수입니다.
- `{{ENTRY_KIND}}` — 항목 유형입니다. 예: Clip, SidePrompt, Memory.
- `{{ENTRY_TITLE}}` — 항목 제목입니다.

사용자 지정 프롬프트가 제대로 동작하지 않으면 **기본값으로 재설정**을 사용하세요.

### 좋은 사용 예

압축은 다음에 적합합니다:

- 긴 클립 항목
- 시간이 지나며 같은 내용을 반복하게 된 사이드 프롬프트 트래커
- 정확하지만 너무 장황한 메모리 항목
- 항상 활성화되어 토큰을 너무 많이 쓰는 항목

다음 용도로는 사용하지 마세요:

- 채팅에서 새 메모리 만들기
- 새 사실 추가
- 항목에 원래 없던 누락된 연속성 고치기
- STMB 밖의 일반 로어북 항목 편집

압축은 정리 도구이지, 메모리 생성 도구가 아닙니다.

전체 기준 문서는 [readme.md](readme.md)를 참고하세요.

대부분의 사용자가 먼저 알아야 할 핵심 설정:

* **메모리 요약 자동 생성**은 자동 기억 생성을 켭니다
* **자동 요약 간격**과 **자동 요약 버퍼**는 자동 생성 시점을 조절합니다
* **Auto-hide/unhide memories**는 토큰 절약에 도움을 줍니다
* **수동 로어북 모드 활성화**와 **로어북이 없으면 자동 생성**은 기억 저장 위치를 정합니다
* **티어가 준비되면 통합을 묻기**는 통합 확인만 표시합니다

## 🔧 문제 해결 (작동하지 않을 때)

전체 문제 해결 문서는 [readme.md](readme.md)를 참고하세요.

가장 빠른 점검 항목:

* STMB가 켜져 있고 확장 메뉴에 **Memory Books**가 보이는지 확인하세요
* auto-summary가 실행되지 않으면 먼저 수동 기억 1개를 만들었는지, interval/buffer가 적절한지 확인하세요
* 기억이 저장되지 않으면 lorebook이 채팅에 연결되어 있는지 또는 **로어북이 없으면 자동 생성**이 켜져 있는지 확인하세요
* Regex 동작이 이상하면 Regex 확장 자체보다 **📐 정규식 구성…** 안의 선택을 먼저 확인하세요
* 통합 확인이 나타나지 않으면 대상 tier와 통합 확인 옵션을 확인하세요
