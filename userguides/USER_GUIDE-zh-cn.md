## 🚀 快速入门（5分钟创建你的第一个记忆）

**刚接触 ST Memory Books？** 下面几步就能让它开始自动记忆：

### 步骤 1：找到扩展
- 找到聊天输入框旁边的魔法棒图标（🪄）
- 点击它，然后点 **“Memory Books”**
- 你会看到 ST Memory Books 的控制面板

### 步骤 2：开启自动魔法
- 在控制面板里找到 **“自动创建记忆摘要”**
- 把它打开
- 将 **“自动摘要间隔”** 先设为 **20-30 条消息**
- 将 **“自动摘要缓冲区”** 先保持在较低值，`0-2` 是不错的起点
- 先手动创建一条记忆，让当前聊天进入可自动处理的状态
- 就这样！🎉

### 步骤 3：正常聊天
- 继续像平常一样聊天
- 当你又发了 20-30 条消息后，ST Memory Books 会自动：
  - 读取上一个已处理检查点之后的新消息
  - 请求 AI 写一条摘要
  - 把它保存到你的记忆集合中
  - 完成后显示通知

**恭喜！** 你现在已经启用了自动记忆管理，不用再担心忘记几章之前发生了什么。

## 💡 ST Memory Books 到底能做什么

把 ST Memory Books 想成你的 **AI 聊天图书管理员**：

### 🤖 **自动摘要**
*“我不想操心，直接让它工作就行”*
- 在后台监控聊天
- 每隔 X 条消息自动创建记忆
- 非常适合长篇角色扮演、创作写作或持续推进的故事

### ✋ **手动创建记忆**
*“我想决定哪些内容被保存”*
- 用简单的箭头按钮（► ◄）标记重要场景
- 需要时按需创建记忆，记录特殊时刻
- 适合捕捉关键剧情点或角色变化

### 📊 **侧边提示词与智能追踪器**
*“我想追踪关系、剧情线或数值”*
- 可复用的提示片段，用来增强记忆生成
- 带有现成追踪器的模板库
- 可自定义 AI 提示，用来追踪任何你想记录的内容
- 自动更新状态板、关系状态、剧情摘要
- 示例：“谁喜欢谁？”、“当前任务状态”、“角色情绪追踪器”

### 📚 **记忆集合**
*所有记忆存放的地方*
- 自动整理且可检索
- 与 SillyTavern 自带 lorebook 系统协同工作
- 你的 AI 可以在新对话里引用过往记忆

3. 如果你想让生成稍微延后，可以再加一点 `自动摘要缓冲区`
4. 先手动创建一条记忆，把当前聊天“点燃”

**你会得到：**
- 不需要手动操作
- 稳定地创建记忆
- 不会错过重要剧情转折
- 单人聊天和群聊都能用

**小建议：** 可以先从 30 条消息开始，再根据你的聊天节奏调整。快节奏聊天可能适合 50+，慢而细的聊天则可能更适合 20。

</details>

<summary><strong>✋ “手动控制” （适合选择性记忆）</strong></summary>

1. 在聊天消息上找到小箭头按钮（► ◄）

**你会得到：**
- 对记忆内容的完全控制
- 很适合捕捉特定时刻
- 对需要明确边界的复杂场景特别有用

**你会得到：**
- 非常快速的记忆创建
- 批量操作
- 与自定义工作流集成

</details>

## ✂️ 摘录到记忆书

当你只想保存一句重要台词或一个事实，而不需要创建完整场景记忆时，使用 **摘录到记忆书**。在聊天中高亮文本，点击浮动剪刀按钮，然后选择现有摘录条目或创建新条目。

不确定该用摘录还是 Side Prompt？请看 [摘录 vs 侧边提示词](#clips-vs-side-prompts)。

### 什么时候该用摘录？

摘录最适合保存你希望 AI 记住的小事实，例如：

- 角色偏好
- 承诺或秘密
- 关系细节
- 宠物、地点、物品或反复出现的细节
- 不需要完整记忆总结的快速“备忘”

如果是较大的场景，请改用正常的记忆创建。

### 摘录如何工作

1. 高亮你想保存的句子或短语。
2. 点击浮动剪刀按钮。
3. 选择一个现有摘录条目，或创建新条目。
4. 检查条目预览。
5. 保存摘录。

摘录条目是普通 lorebook 条目，但会标记为 `[STMB Clip]`。例如：

```txt
Seraphina Healed Me [STMB Clip]
```

在条目内部，STMB 会用清晰的区段格式保存内容：

```md
=== Seraphina Healed Me ===

- Seraphina healed my wounds with magic.

=== END Seraphina Healed Me ===
```

### 创建或重命名摘录条目

创建新摘录条目时，条目标题也会成为区段标题。你可以在摘录时重命名条目，STMB 会同步更新区段标题。

新的摘录条目可以是：

- **常驻启用**：适合应该始终可用的事实
- **关键词触发**：适合只在相关词出现时才需要的事实

当摘录只和某个主题、角色、地点、宠物、物品或关系有关时，使用关键词更合适。

### 浮动剪刀按钮

浮动剪刀按钮只会在你高亮聊天内文本后出现。你可以在 Memory Books 主弹窗中打开或关闭这个按钮。

### 检查较长的摘录条目

如果摘录条目变长，STMB 可能会提醒你检查它。你可以自己编辑，也可以使用 **压缩**，让 AI 把摘录、Side Prompt 或 STMB 记忆条目改写得更节省 token，然后再决定是否替换原文。

## ✂️ 摘录 vs 侧边提示词

摘录和 Side Prompts 都会把信息保存到你的记忆书里，但它们不是同一种用途。

简单规则：**摘录保存一个具体事实。Side Prompt 维护一个会变化的追踪器。**

| **摘录** | **Side Prompts** |
|---|---|
| 把选中的聊天文本保存到记忆书条目中。 | 请求 AI 阅读聊天并更新追踪器条目。 |
| 最适合一个明确事实、台词、承诺、偏好、物品或笔记。 | 最适合会随时间变化的信息，例如关系状态、任务进度、库存或未解决剧情线。 |
| 你选择确切文本，STMB 保存你选中的内容。 | AI 会解读聊天，然后写入或更新追踪器。 |
| 当事实已经很明确、不需要分析时使用。 | 当 AI 需要比较、总结或根据多条消息更新状态时使用。 |
| 通常只会在你手动添加另一条摘录时增长。 | 可以随着故事变化反复更新。 |
| 可以理解为：“把这条笔记钉住。” | 可以理解为：“让这个区块保持更新。” |

适合做摘录的例子：

- `Aiko 喜欢蜂蜜茶。`
- `Theron 承诺不再对她撒谎。`
- `Colt 叫她 Boss。`

适合做 Side Prompt 的例子：

- 关系状态
- 当前任务进度
- 库存与资源
- NPC 名录
- 未解决剧情线

如果你只需要记住一个细节，用摘录。如果你需要持续更新的追踪器，用 Side Prompt。

## 🔎 主题摘录

主题摘录用于从您已经创建的记忆中，制作一条聚焦的“关于这个主题”的记忆条目。

可以把它理解为让 STMB：

> “阅读我保存的记忆，然后为这个人物、地点、关系、剧情线、物品、秘密或主题创建一条有用的条目。”

它仍然是摘录式条目，但您不是在摘录选中的聊天文本。STMB 会使用现有记忆条目作为来源。

简单规则：**摘录保存选中的文本。主题摘录从已保存记忆中整理相关细节。Side Prompt 持续维护追踪器。**

### 什么时候使用主题摘录

当您的记忆书中已经有多条记忆，并且您想要一条更容易触发的特定主题条目时，使用主题摘录。

适合的例子：

- 反复出现的 NPC
- 两个角色之间的关系
- 一个谜团或调查
- 一个地点
- 一个派系
- 角色的能力、伤势、承诺、秘密或偏好
- 跨多个场景出现的剧情线

主题示例：

```txt
Seraphina
{{user}} 的魔法
Alex 和 Mira 的关系
Black Harbor 调查
银钥匙
```

### 什么时候不要使用主题摘录

不要在这些情况下使用主题摘录：

- 您只想保存聊天中选中的一句话 — 使用**摘录到记忆书**
- 您想要一个在未来记忆运行中自动更新的追踪器 — 使用 **Side Prompt**
- 您想缩短一条过长条目 — 使用**压缩**
- 您想把多条记忆合并成更高层级的总结 — 使用**总结整合**

### 如何使用主题摘录

1. 打开 Memory Books 弹窗。
2. 点击 **🔎 主题摘录**。
3. 选择**来源记忆书**。
4. 输入**主题**。
   - 这是 AI 应该聚焦的对象。
   - 尽量具体。
5. 输入**关键词**。
   - 这些会成为世界书条目的激活关键词。
   - 如果留空，STMB 会使用主题。
6. 选择模式：
   - **创建新的主题摘录**会创建新的 `[STMB Clip]` 条目。
   - **更新现有条目**会更新现有摘录条目。
7. 选择**生成配置文件**。
   - 这决定由哪个 AI 连接/模型来撰写草稿。
8. 可选：如果想更改发送给 AI 的说明，点击**编辑主题摘录提示**。
9. 点击**生成草稿**。
10. 检查生成的草稿。
11. 如有需要，编辑草稿。
12. 点击**保存主题摘录**。

STMB 不会自动保存草稿。只有点击**保存主题摘录**后，世界书才会更改。

### 创建新的主题摘录

创建新的主题摘录时，STMB 会创建一条摘录式世界书条目。

例如，如果您的主题是：

```txt
Seraphina
```

条目标题会类似：

```txt
关于 Seraphina [STMB Clip]
```

条目中的可见部分会使用与普通摘录条目相同的摘录包装格式。

### 更新现有主题摘录

主题摘录也可以更新现有的 `[STMB Clip]` 条目。

如果您已经有这样的条目：

```txt
关于 Seraphina [STMB Clip]
```

而上次更新之后又新增了记忆，这会很有用。

主题摘录更新成功保存后，STMB 会在该条目上保存一小段运行历史。它包含本次运行使用过的来源记忆。下一次更新时，STMB 可以利用这段历史，只查找新的或已更改的来源记忆，而不是重新读取所有内容。

这样可以让更新更小，也能避免反复把相同的旧记忆发送给 AI。

### 从所有来源记忆重建

更新现有主题摘录时，您可能会看到**从所有来源记忆重建**。

普通更新时请保持关闭。STMB 会尽可能只使用新的或已更改的来源记忆。

在这些情况下开启：

- 现有主题摘录已经严重过时
- 您更改了主题摘录提示
- 您大幅更改了主题或关键词
- 您想让 AI 重新考虑该主题的所有已保存记忆
- 该条目还没有有用的运行历史

### 它使用哪些来源条目？

主题摘录使用所选记忆书中已确认的 STMB 记忆条目。

它不会使用：

- 普通摘录条目
- Side Prompt 追踪器条目
- 不由 STMB 管理的普通世界书条目

这样可以让主题摘录专注于 STMB 能安全识别的记忆。

### 使用主题摘录的好习惯

使用聚焦的主题。

较好：

```txt
Alex 和 Mira 的关系
```

不太有用：

```txt
关于整个故事的一切
```

较好：

```txt
银钥匙
```

不太有用：

```txt
重要物品
```

主题越具体，AI 越容易判断哪些内容属于这个主题，哪些不属于。

### 编辑提示

主题摘录提示可以编辑。

默认提示会要求 AI：

- 只提取与主题相关的信息
- 避免无关事件
- 保留姓名、关系、偏好、承诺、秘密、限制和未解决问题
- 发现冲突时说明冲突，而不是悄悄选择一个版本
- 更新现有摘录内容，但不要重复
- 不要编造缺失细节

提示必须包含：

```txt
{{SOURCE_MEMORIES}}
```

没有这个占位符，STMB 就不知道应该把来源记忆放在哪里。

其他支持的占位符包括：

```txt
{{MODE}}
{{TOPIC}}
{{KEYWORDS}}
{{EXISTING_CLIP}}
{{EXISTING_ENTRY_CONTENT}}
{{SOURCE_MEMORIES}}
```

如果您的自定义提示不再好用，请使用 **Reset to Default**。

## 🙈 节省 token：隐藏 / 显示消息

在长聊天里，最简单的减轻负担和省 token 的方法之一，就是在消息已经转成记忆后把它们隐藏起来。

### “隐藏”是什么意思？

隐藏消息 **不会删除** 它们。它只是让 AI 看不到这些消息。你的聊天记录仍然存在，记忆也仍然保留在 lorebook 里，所以重要信息不会丢失，只是不再直接发送给 AI。

### 为什么要用它？

隐藏 / 重新显示 在以下情况非常有用：
- 你的聊天已经很长
- 这些消息已经被写进记忆

### 创建记忆后自动隐藏

STMB 可以在记忆创建后自动隐藏消息。你可以选择：

- **不自动隐藏**：保持全部可见，你可以手动用 `/hide x-y` 隐藏
- **自动隐藏到最后一条记忆为止的所有消息**：隐藏已经被记忆覆盖的内容
- **只自动隐藏最后一条记忆中的消息**：只隐藏最近处理过的范围

你还可以通过 **保留未隐藏的消息数量** 控制保留多少最近消息可见。

### 生成记忆前重新显示

**生成记忆前取消隐藏的消息 (运行 /unhide X-Y)** 会在生成记忆前，临时对所选范围执行 `/unhide X-Y`。如果你经常重做记忆，这个很有用。

### 新手推荐设置

Aiko 的设置：
- 使用 **自动隐藏所有消息直到最后一个记忆**
- 保留 **2** 条消息不隐藏
- 打开 **生成记忆前取消隐藏的消息 (运行 /unhide X-Y)**

## 🧭 压缩 vs 总结整合

名字有点像，但它们做的是不同工作。

简单规则：**压缩清理一个条目。总结整合把多个记忆合并成更高层级的回顾。**

| **压缩** | **总结整合** |
|---|---|
| 让一个现有的 STMB 管理条目变短。 | 把多个记忆或总结合并成一个更高层级的回顾。 |
| 一次处理一个摘录、Side Prompt 条目或 STMB 记忆条目。 | 从多个已选择的记忆 / 总结条目生成。 |
| 适合条目仍然有用，但太长、重复或占用太多上下文的时候。 | 适合旧场景记忆堆积起来，应该变成 Arc、Chapter、Book、Legend、Series 或 Epic 总结的时候。 |
| 以更节省 token 的形式重写所选条目。 | 从所选源条目创建一个新的总结条目。 |
| 应该保留现有事实，并移除臃肿内容。 | 应该保留更大的连续性脉络，并减少逐场景细节。 |
| 不会从原始聊天创建新记忆。 | 不会单独压缩某个臃肿条目。 |
| 可以理解为：“修剪这一个条目。” | 可以理解为：“把这些记忆卷成一条回顾。” |

两个工具都需要先审阅。STMB 会先显示 AI 写出的内容，再保存或替换任何东西。

## 🌈 总结整合

总结整合会把较早的 STMB 记忆压缩成更高层级的回顾条目，让长篇故事更容易维护。

### 问：什么是总结整合？

**答：** 它不是一直只做场景级记忆。STMB 可以把现有记忆或摘要合并成更紧凑的回顾。第一层是 **Arc**，更高层级也支持更长篇幅的故事：

- Arc
- Chapter
- Book
- Legend
- Series
- Epic

### 问：为什么要用它？

**答：** 在这些情况下特别有用：

- 记忆列表越来越长
- 老条目不再需要逐场景细节
- 你想降低 token 消耗但保留连续性
- 你想得到更干净、更高层的叙事回顾

### 问：它会自动运行吗？

**答：** 不会。总结整合仍然需要你确认。

- 你可以随时从主面板手动打开 **整合记忆**
- 你也可以启用 **当某个层级准备好时提示合并**
- 当某个目标层级达到其已保存的最小合格数量时，STMB 会显示一个 **yes/later** 确认框
- 选择 **Yes** 只会打开已选中该层级的整合面板，不会悄悄自动执行

### 问：我该怎么用？

**答：** 创建整合摘要的步骤是：

1. 在 STMB 主面板中点击 **整合记忆**
2. 选择目标摘要层级
3. 选择想纳入的来源条目
4. 视需要决定是否在新摘要创建后停用来源条目
5. 点击 **Run**

如果你想预览这些条目，请在偏好设置里启用预览。

## 🎨 追踪器、侧边提示词与模板（高级功能）

**侧边提示词** 是后台追踪器，能帮你维持持续变化的故事信息。它们会在记忆创建之外运行，并随着时间更新独立的侧边提示词故事书条目。你可以把它们理解成 **一边看着故事、一边帮你把关键细节保持最新的助手**。

如果你只想保存一个高亮的固定事实，请使用 [摘录到记忆书](#clip-to-memory-book)。Side Prompts 更适合重复或持续追踪的信息。

### 🚀 **用模板快速开始**

1. 打开 Memory Books 设置
2. 点击 **侧边提示词**
3. 浏览 **template library**，选择适合你故事的模板：

   * **Character Development Tracker** - 追踪人格变化和成长
   * **Relationship Dynamics** - 追踪角色关系
   * **Plot Thread Tracker** - 追踪持续推进的剧情线
   * **Mood & Atmosphere** - 追踪情绪基调
   * **World Building Notes** - 追踪设定细节和世界观
4. 启用你想要的模板（之后还可以继续自定义）
5. 如果模板使用自动触发，STMB 会在记忆创建时一并更新这个 side-prompt 条目

[Scribe 逐步演示如何启用 Memory Books 的侧边提示词](https://scribehow.com/viewer/How_to_Enable_Side_Prompts_in_Memory_Books__fif494uSSjCmxE2ZCmRGxQ)

### ⚙️ **侧边提示词的工作方式**

* **后台追踪器**：它们静静运行，并随时间更新信息
* **不侵入**：不会改动你的主 AI 设置或角色提示词
* **按聊天控制**：不同聊天可以使用不同追踪器
* **基于模板**：可以用内置模板，也可以自己创建
* **自动或手动**：标准模板可以自动运行；带自定义运行时宏的模板只能手动运行
* **宏支持**：`Prompt`、`Response Format`、`Title` 和关键词字段可以展开标准 ST 宏，例如 `{{user}}` 和 `{{char}}`
* **运行时宏**：非标准的 `{{...}}` token 会变成命令输入，例如 `{{npc name}}="Jane Doe"`
* **允许纯文本**：侧边提示词不一定要返回 JSON
* **覆盖行为**：侧边提示词会持续更新自己的追踪条目，而不是每次都新建一条顺序记忆

### 🛠️ **管理侧边提示词**

* **侧边提示词管理器**：创建、编辑、复制和整理追踪器
* **启用 / 禁用**：随时开关追踪器
* **导入 / 导出**：共享模板或备份
* **状态视图**：查看当前聊天里哪些追踪器在运行，以及它们何时触发
* **安全检查**：如果模板包含自定义运行时宏，STMB 会在保存 / 导入时移除自动触发并显示警告提示

### 💡 **模板示例**

* Side Prompt Template Library（导入这个 JSON）：
  [SidePromptTemplateLibrary.json](/resources/SidePromptTemplateLibrary.json)

示例提示想法：

* “追踪重要对话和角色互动”
* “保持当前任务状态最新”
* “记录出现时的新世界观细节”
* “追踪角色 A 和角色 B 之间的关系”

### 🔧 **创建自定义侧边提示词**

1. 打开侧边提示词管理器
2. 点击 **新建**
3. 写一条简短、明确的指令
   （例如：“始终记录每个场景里的天气情况”）
4. 可以选填标准 ST 宏，比如 `{{user}}` 或 `{{char}}`
5. 如果加入自定义运行时宏，例如 `{{location name}}`，就需要用 `/sideprompt "Name" {{location name}}="value"` 手动运行
6. 保存并启用它
7. 如果该追踪器使用自动触发，它就会随着时间自动更新这类信息；否则就在需要时手动运行

### 💬 **小技巧**

侧边提示词 最适合做成 **小而聚焦** 的内容。
与其写“追踪一切”，不如写“追踪主角之间的恋爱张力”。

### ⌨️ **手动 /sideprompt 语法**

使用：
`/sideprompt "Name" {{macro}}="value" [X-Y]`

示例：
- `/sideprompt "Status" 10-20`
- `/sideprompt "NPC Directory" {{npc name}}="Jane Doe" 40-50`
- `/sideprompt "Location Notes" {{place name}}="Black Harbor" 100-120`

注意：

- 侧边提示词名称必须加引号
- 运行时宏的值也必须加引号
- 选完侧边提示词后，Slash 命令自动补全会提示所需的运行时宏
- 如果模板包含自定义运行时宏，STMB 会把它保留为仅手动运行，并移除自动触发
- `X-Y` 是可选的。如果你不写范围，STMB 会使用自上次该 side prompt 更新以来的消息
- 如果你是手动并分开运行侧边提示词，别忘了打开 **生成前取消隐藏**

### 🧠 使用 Regex 扩展进行高级文本控制

**想完全控制 STMB 发送给 AI 的文本和接收回来的文本？** STMB 可以在生成前和保存前运行你选定的 Regex 脚本。

这在以下情况下很有用：
- 清理 AI 回应里的重复废话
- 在生成前统一名称或术语
- 在 STMB 解析或预览前重新格式化文本

#### **现在它是怎么工作的**

1. 在 SillyTavern 的 **Regex** 扩展里创建你想要的脚本
2. 在 STMB 里开启 **使用正则表达式（高级）**
3. 点击 **📐 配置正则表达式…**
4. 选择 STMB 应该运行哪些脚本：
   - 在把文本发送给 AI 之前
   - 在把响应加入 lorebook 之前

#### **重要行为**

- STMB 的 Regex 选择由 **STMB 本身**控制，不由 Regex 扩展里脚本的启用 / 禁用状态控制
- 在 STMB 里选中的脚本，即使在 Regex 扩展里被禁用，也仍然可以运行
- STMB 支持出站和入站处理的多选

#### **快速示例**

如果你的模型总是在摘要后面加 `(OOC: I hope this summary is helpful!)`，你可以：

1. 创建一个删除这段文字的 Regex 脚本
2. 在 STMB 里打开 **使用正则表达式（高级）**
3. 打开 **📐 配置正则表达式…**
4. 把那个脚本加入 **incoming** 选择

这样 STMB 就会在预览或保存前清理响应内容。

## 🧹 压缩

压缩适用于某个由 STMB 管理的 lorebook 条目仍然有用、但已经变得太长或重复的情况。你不用手动删改，可以让 AI 把这个条目重写成更节省 token 的版本。

不确定该用压缩还是总结整合？参考上面的简单规则：**压缩清理一个条目。总结整合把多个记忆合并成更高层级的回顾。**

这是一个 **先审阅** 的工具。STMB 会在替换任何内容之前显示原始内容和压缩草稿。

### 什么可以被压缩？

压缩可以列出所选记忆书中的这些条目：

- 摘录条目
- 侧边提示词追踪条目
- STMB 记忆条目

它不会显示不由 STMB 管理的普通 lorebook 条目。

### 如何使用压缩

1. 打开 Memory Books 弹窗。
2. 点击 **📝 压缩**。
3. 选择你要检查的 **记忆书**。如果当前聊天已经有记忆书，它可能会自动选中。
4. 选择一个 **压缩配置文件**。这会决定由哪个 AI 连接 / 模型重写条目。
5. 可选：如果想修改重写指令，请点击 **编辑压缩提示**。
6. 在表格中找到条目，然后点击 **压缩条目**。
7. 检查结果：
   - **原始内容** 显示当前已保存的内容。
   - **压缩草稿** 显示 AI 重写结果。
   - 两者都会显示估算 token 数。
8. 根据需要编辑压缩草稿。
9. 选择一个操作：
   - **替换为压缩版本**：把草稿保存并覆盖原条目。
   - **复制压缩草稿**：只复制草稿，不保存。
   - **取消**：保持条目不变。

STMB 不应该静默替换原文。如果你没有点击 **替换为压缩版本**，lorebook 条目会保持原样。

### 编辑压缩提示

压缩提示控制 AI 如何重写条目。内置提示刻意偏保守：保留重要事实、姓名、代词、宏、包装标题和结尾标记；移除重复和低价值措辞；不要编造内容。

此提示支持这些占位符：

- `{{ENTRY_CONTENT}}` — 当前条目内容。此项必需。
- `{{ENTRY_KIND}}` — 条目类型，例如 Clip、SidePrompt 或 Memory。
- `{{ENTRY_TITLE}}` — 条目标题。

如果自定义提示开始表现不好，请使用 **重置为默认值**。

### 适合的用途

适合用压缩处理：

- 较长的摘录条目
- 逐渐重复自己的侧边提示词追踪器
- 内容正确但臃肿的记忆条目
- 一直激活、正在消耗过多 token 的条目

不要用它来：

- 从聊天中创建新记忆
- 添加新事实
- 修补条目里原本不存在的连续性
- 编辑 STMB 之外的普通 lorebook 条目

压缩是清理工具，不是记忆生成工具。

多数用户最先应该了解这些控制项：

## 🔧 疑难解答（当事情不工作时）

这不是完整的排障清单。完整列表请看 [readme-zh-cn.md](readme-zh-cn.md)。

先做这些最常见的检查：

- 确认 STMB 已启用，而且扩展魔杖菜单下出现了 **Memory Books**
- 如果自动摘要没有触发，先确认你已经手动创建过一条记忆，并检查 interval / buffer 是否合理
- 如果记忆无法保存，请确认聊天已经绑定 lorebook，或启用了 **Auto-create lorebook if none exists**
- 如果记忆没有触发，确认 `delay until recursion` 已关闭
- 如果 regex 行为看起来不对，请检查 **📐 配置正则表达式…** 里的选择，而不只是看 Regex 扩展本身
- 如果整合没有弹出，请确认 **当某个层级准备好时提示合并** 已启用，并且目标层级包含在 **自动合并层级** 中
