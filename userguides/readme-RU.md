# 📕 Memory Books (расширение для SillyTavern)

Расширение нового поколения для SillyTavern, предназначенное для автоматического, структурированного и надежного создания памяти. Отмечайте сцены в чате, генерируйте JSON-сводки с помощью ИИ и сохраняйте их как записи в ваших лорбуках. Поддерживает групповые чаты, расширенное управление профилями, Side Prompts/трекеры и многоуровневую консолидацию памяти.

### ❓ Словарь
- Scene (сцена) → Memory (воспоминание)  
- One saved fact (один сохранённый факт) → Clip  
- Ongoing tracker (постоянный трекер) → Side Prompt  
- Many Memories (много воспоминаний) → Summary / Consolidation (сводка / консолидация)  
- One long entry (одна длинная запись) → Compaction (сжатие)

### Clips vs Side Prompts

<details>
<summary><strong>Clips vs Side Prompts</strong></summary>

| **Clips** | **Side Prompts** |
|---|---|
| Сохраняют выделенный текст из чата в запись Memory Book. | Просят ИИ просмотреть чат и обновить запись-трекер. |
| Лучше подходят для одного ясного факта, реплики, обещания, предпочтения, предмета или заметки. | Лучше подходят для информации, которая меняется со временем. |
| Проще говоря: «закрепить эту заметку». | Проще говоря: «поддерживать этот раздел в актуальном состоянии». |

</details>

Более подробное объяснение см. в [Руководстве пользователя](USER_GUIDE-RU.md#-clips-vs-side-prompts).

### Compaction vs Consolidation

<details>
<summary><strong>Compaction vs Consolidation</strong></summary>

| **Compaction / Сжатие** | **Consolidation / Консолидация** |
|---|---|
| Сокращает одну существующую запись, которой управляет STMB. | Объединяет несколько воспоминаний или сводок в обзор более высокого уровня. |
| Используйте, когда запись Clip, Side Prompt или Memory полезна, но стала слишком длинной. | Используйте, когда несколько воспоминаний готовы стать Arc, Chapter, Book или другой крупной сводкой. |
| Проще говоря: «обрезать эту одну запись». | Проще говоря: «свернуть эти воспоминания в обзор». |

</details>

Более подробное объяснение см. в [Руководстве пользователя](USER_GUIDE-RU.md#-compaction-vs-consolidation).

## 📋 Предварительные требования

- **SillyTavern:** 1.14.0+ (рекомендуется последняя версия).
- **Опциональная очередь задач:** STMB работает и без очереди задач. Чтобы использовать очередь, установите и включите **Chat Top Bar** / **Chat Top Info Bar** — официальное расширение SillyTavern, которое добавляет верхнюю панель в окно чата. STMB использует эту панель для кнопки и боковой панели **Задачи Книг Памяти**.
- **Поддержка Chat Completion:** полная поддержка OpenAI, Claude, Anthropic, OpenRouter или другого Chat Completion API.
- **Поддержка Text Completion:** Text Completion API (Kobold, TextGen и т. п.) поддерживаются, если они подключены через Chat Completion API endpoint, совместимый с OpenAI. Рекомендуется настроить подключение Chat Completion API согласно советам для KoboldCpp ниже (при необходимости адаптируйте для Ollama или другого ПО). После этого настройте профиль STMB и используйте Custom (рекомендуется) или полностью ручную конфигурацию (только если Custom не работает или у вас больше одного custom-подключения).

**ПРИМЕЧАНИЕ:** если вы используете Text Completion, у вас обязательно должен быть Chat Completion Preset.

### Советы по использованию 📕 ST Memory Books с KoboldCpp

Настройте это в ST. Вы сможете переключиться обратно на Text Completion после того, как убедитесь, что STMB работает.

- Chat Completion API
- Custom chat completion source
- endpoint `http://localhost:5001/v1` (также можно использовать `127.0.0.1:5000/v1`)
- введите что угодно в `custom API key` (значение не важно, но ST требует ключ)
- Model ID должен быть в формате `koboldcpp/modelname` (не добавляйте `.gguf` в имя модели)
- скачайте и импортируйте любой Chat Completion Preset, чтобы он у вас был. Это предотвращает ошибки `not supported`.
- установите max response length в Chat Completion Preset минимум на 2048; рекомендуется 4096. При меньшем значении ответ может обрезаться.

### Советы по использованию 📕 ST Memory Books с Llama.cpp

Как и с Kobold, настройте это как _Chat Completion API_ в ST. После проверки работы STMB можно переключиться обратно.

- Создайте новый connection profile для Chat Completion API.
- Completion Source: `Custom (Open-AI Compatible)`.
- Endpoint URL: `http://host.docker.internal:8080/v1`, если ST запущен в Docker; иначе `http://localhost:8080/v1`.
- Custom API key: введите что угодно (ST требует ключ).
- Model ID: `llama2-7b-chat.gguf` или имя вашей модели; если в llama.cpp запущена только одна модель, это не критично.
- Prompt post-processing: none.

Для запуска Llama.cpp удобно поместить что-то подобное в shell-скрипт или bat-файл:

```sh
llama-server -m <model-path> -c <context-size> --port 8080
```

## 💡 Рекомендуемые глобальные настройки активации World Info/Lorebook

- **Match Whole Words:** оставить выключенным (false).
- **Scan Depth:** чем выше, тем лучше (у меня стоит 8).
- **Max Recursion Steps:** 2 (общая рекомендация, не обязательное требование).
- **Context %:** 80% (на основе контекстного окна 100 000 токенов) — предполагается, что у вас нет очень тяжелой истории чата или ботов.
- Дополнительно: если лорбук памяти — ваш единственный лорбук, убедитесь, что в профиле STMB отключено `Delay until recursion`, иначе воспоминания не будут срабатывать.

## 🧩 Типы памяти: сцены и сводки

📕 Memory Books поддерживает **воспоминания о сценах** и **многоуровневую консолидацию сводок**. Они предназначены для разных типов непрерывности.

### 🎬 Воспоминания о сценах (по умолчанию)

Воспоминания о сценах фиксируют, **что произошло** в конкретном диапазоне сообщений.

- Основаны на явном выборе сцены (► ◄).
- Хорошо подходят для моментального, детального припоминания.
- Сохраняют диалог, действия и немедленные последствия.
- Лучше всего работают при частом использовании.

Это стандартный и наиболее часто используемый тип памяти.

---

### 🌈 Консолидация сводок

![Кнопка консолидации](https://github.com/aikohanasaki/imagehost/blob/main/STMemoryBooks/button-consolidate.png)


Консолидация сводок фиксирует, **что изменилось со временем** в нескольких воспоминаниях или сводках.

Вместо пересказа одной сцены консолидированные сводки фокусируются на:

- развитии персонажей и сдвигах в отношениях;
- долгосрочных целях, напряжении и разрешении конфликтов;
- эмоциональной траектории и направлении повествования;
- устойчивых изменениях состояния, которые должны оставаться стабильными.

Первый уровень консолидации — **Arc**, который строится из воспоминаний о сценах. Для более длинных историй поддерживаются более высокие уровни:

- Arc
- Chapter
- Book
- Legend
- Series
- Epic

> 💡 Думайте о них как о *кратких пересказах*, а не как о журналах сцен.

#### Когда использовать консолидированные сводки

- После крупного сдвига в отношениях.
- В конце главы или арки истории.
- Когда меняются мотивации, доверие или динамика власти.
- Перед началом новой фазы истории.

#### Как это работает

- Консолидированные сводки создаются из существующих воспоминаний/сводок STMB, а не напрямую из сырого чата.
- Инструмент **Consolidate Memories** позволяет выбрать целевой уровень сводки и исходные записи.
- STMB может отслеживать выбранные уровни сводок и показывать подтверждение yes/later, когда уровень достигает сохраненного минимального числа подходящих исходных записей.
- STMB может отключить исходные записи после консолидации, если вы хотите, чтобы сводка более высокого уровня взяла на себя их роль.
- Неудачные ответы ИИ можно просмотреть и исправить в интерфейсе перед повторной попыткой сохранения.

Это дает:

- меньшее использование токенов;
- лучшую непрерывность повествования в длинных чатах.

## 📝 Генерация памяти

### **Только JSON на выходе**

Все промты и пресеты **должны** инструктировать ИИ возвращать только валидный JSON, например:

```json
{
  "title": "Короткое название сцены",
  "content": "Подробная сводка сцены...",
  "keywords": ["ключевоеслово1", "ключевоеслово2"]
}
```

**В ответе не должно быть никакого другого текста.**

### **Встроенные пресеты**

1. **Summary:** подробные пошаговые (beat-by-beat) сводки.
2. **Summarize:** Markdown-заголовки для timeline, beats, interactions и outcome.
3. **Synopsis:** подробный, структурированный markdown.
4. **Sum Up:** краткая beat-сводка с timeline.
5. **Minimal:** сводка в 1–2 предложения.
6. **Northgate:** литературный стиль сводки для creative writing.
7. **Aelemar:** фокус на сюжетных точках и воспоминаниях персонажей.
8. **Comprehensive:** сводка в стиле synopsis с улучшенным извлечением ключевых слов.

### **Пользовательские промты**

- Можно создавать собственные промты, но они **должны** возвращать валидный JSON, как показано выше.

## 📚 Интеграция с лорбуком

- **Автоматическое создание записей:** новые воспоминания сохраняются как записи со всеми метаданными.
- **Распознавание по флагу:** только записи с флагом `stmemorybooks` распознаются как воспоминания.
- **Автонумерация:** последовательная нумерация с ведущими нулями, поддерживаются разные форматы (`[000]`, `(000)`, `{000}`, `#000`).
- **Ручной/автоматический порядок:** настройки порядка вставки для каждого профиля.
- **Обновление редактора:** опционально автоматически обновляет редактор лорбука после добавления воспоминания.

> **Существующие воспоминания нужно конвертировать!**  
> Используйте [Lorebook Converter](../resources/lorebookconverter.html), чтобы добавить флаг `stmemorybooks` и обязательные поля.

## ✂️ Clip to Memory Book

![Клип текста](https://github.com/aikohanasaki/imagehost/blob/main/STMemoryBooks/clip.png)


**Clip to Memory Book** нужен для быстрых заметок «запомни это». Выделите важный текст в чате, нажмите плавающую кнопку с ножницами и сохраните выбранный текст как пункт в Memory Book без предварительного открытия редактора lorebook.

Если вам нужен постоянный трекер, который обновляется со временем, используйте Side Prompt. Короткое правило: **Clip = один сохранённый факт; Side Prompt = постоянный трекер.**

#### Как это работает
- Выделите точный текст, который нужно запомнить.
- Нажмите плавающую кнопку с ножницами. Эту кнопку можно включить или выключить в окне Memory Books.
- Выберите существующую запись Clip или создайте новую.
- Перед сохранением проверьте текущую запись и обновлённый предпросмотр.
- При необходимости переименуйте запись/раздел.

Записи Clip — это обычные записи lorebook с пометкой `[STMB Clip]` в конце названия записи. Например:

```txt
Seraphina Healed Me [STMB Clip]
```

Видимый раздел внутри записи использует название без `[STMB Clip]`:

```md
=== Seraphina Healed Me ===

- Seraphina healed my wounds with magic.
- Seraphina, guardian of this forest

=== END Seraphina Healed Me ===
```

#### Советы
- Одна запись Clip содержит один раздел. Используйте сфокусированные названия вроде `Things {{user}} Likes`, `Pet Names` или `Food Preferences`, чтобы ключевые слова оставались точными.
- Новые записи Clip могут быть всегда активными или активироваться по ключевым словам. Всегда активный режим проще; ключевые слова лучше, если запись должна появляться только иногда.
- Существующие записи можно превратить в записи Clip, добавив `[STMB Clip]` в конец названия.
- Для длинных записей Clip может появиться напоминание проверить или сжать запись. Сжатие помогает сделать записи Clip, Side Prompt и Memory более экономными по токенам перед заменой оригинала.
- Записи Clip не добавляют атрибуцию источника. Они сохраняют только тот текст, который вы выбрали.

## 🔎 Тематический фрагмент

Тематический фрагмент создаёт или обновляет сфокусированную запись памяти в стиле Clip об одной теме.

Используйте его, когда у вас уже есть сохранённые STMB-воспоминания, но нужна одна аккуратная запись «об этой теме», которая собирает связанные детали из этих воспоминаний. Например:

- `О Seraphina`
- `О магии {{user}}`
- `Об отношениях Alex и Mira`
- `О расследовании Black Harbor`

Тематический фрагмент отличается от обычного Clip to Memory Book. Обычный Clip напрямую сохраняет выделенный текст из чата. Тематический фрагмент читает существующие записи памяти STMB, просит ИИ извлечь детали по одной теме, а затем показывает редактируемый черновик перед сохранением.

#### Как это работает

1. Откройте Memory Books.
2. Нажмите **🔎 Тематический фрагмент**.
3. Выберите **Исходную Книгу Памяти**.
4. Введите **Тему**.
5. Введите **Ключевые слова** активации или оставьте поле пустым, чтобы использовать тему.
6. Выберите, создать новый тематический фрагмент или обновить существующую запись `[STMB Clip]`.
7. Выберите **Профиль генерации**.
8. Нажмите **Сгенерировать черновик**.
9. Проверьте и отредактируйте черновик.
10. Нажимайте **Сохранить тематический фрагмент** только когда результат вас устраивает.

Тематический фрагмент сохраняет записи как обычные Clip-записи с пометкой `[STMB Clip]`. Новые записи используют заголовок вроде:

```txt
О Seraphina [STMB Clip]
```

#### Обновление существующих тематических фрагментов

Когда вы обновляете существующий тематический фрагмент, STMB запоминает, какие исходные воспоминания использовались во время последнего успешного запуска. Следующее обновление обычно использует только новые или изменённые исходные воспоминания.

Если вы хотите пересобрать всю запись из всех подходящих воспоминаний, включите **Пересобрать из всех исходных воспоминаний** перед генерацией черновика.

#### Примечания

- Тематический фрагмент использует в качестве источника только подтверждённые записи памяти STMB.
- Clip-записи и записи Side Prompt не используются как исходные воспоминания.
- Целями обновления являются существующие записи `[STMB Clip]`.
- Черновик ИИ всегда можно проверить и отредактировать перед сохранением.
- STMB не сохраняет сгенерированный черновик, пока вы не нажмёте **Сохранить тематический фрагмент**.
- Если запрос большой, STMB может показать предупреждение о токенах перед запуском.

## 🆕 Слэш-команды

- `/creatememory` — создать воспоминание из отмеченной сцены.
- `/scenememory X-Y` — установить диапазон сцены и создать воспоминание, например `/scenememory 10-15`.
- `/nextmemory` — создать воспоминание от конца последнего воспоминания до текущего сообщения.
- `/stmb-catchup interval:x start:y end:y` — создать догоняющие воспоминания для существующего длинного чата, обрабатывая выбранный диапазон сообщений блоками указанного размера.
- `/sideprompt "Name" {{macro}}="value" [X-Y]` — запустить Side Prompt (`{{macro}}` необязательны).
- `/sideprompt-set "Set Name" [X-Y]` — запустить сохраненный Side Prompt Set.
- `/sideprompt-macroset "Set Name" {{macro}}="value" [X-Y]` — запустить Side Prompt Set и передать переиспользуемые значения макросов.
- `/sideprompt-on "Name" | all` — включить Side Prompt по имени или включить все.
- `/sideprompt-off "Name" | all` — выключить Side Prompt по имени или выключить все.
- `/stmb-highest` — вернуть самый высокий message id среди обработанных воспоминаний в этом чате.
- `/stmb-set-highest <N|none>` — вручную установить самый высокий обработанный message id для этого чата.
- `/stmb-stop` — остановить всю активную генерацию STMB везде. Это аварийная остановка.

### `/stmb-catchup`

Используйте `/stmb-catchup`, когда нужно преобразовать существующий длинный чат в воспоминания STMB.

Синтаксис: `/stmb-catchup interval:x start:y end:y`

Пример: `/stmb-catchup interval:30 start:0 end:300`

## 🧭 Режимы работы

### **Автоматический режим (по умолчанию)**

![Пример привязки лорбука к чату](https://github.com/aikohanasaki/imagehost/blob/main/STMemoryBooks/chatlorebook.png)


- **Как это работает:** автоматически использует лорбук, привязанный к текущему чату.
- **Лучше всего для:** простоты и скорости. Большинству пользователей стоит начинать отсюда.
- **Как использовать:** убедитесь, что лорбук выбран в выпадающем меню `Chat Lorebooks` для вашего персонажа или группового чата.


### **Режим авто-создания лорбука**

- **Как это работает:** автоматически создает и привязывает новый лорбук, если лорбука нет, используя ваш шаблон имени.
- **Лучше всего для:** новых пользователей и быстрой настройки. Подходит для создания лорбука в один клик.
- **Как использовать:**
  1. Включите `Auto-create lorebook if none exists` в настройках расширения.
  2. Настройте шаблон имени (по умолчанию: `LTM - {{char}} - {{chat}}`).
  3. Если вы создаете воспоминание без привязанного лорбука, новый лорбук будет создан и привязан автоматически.
- **Плейсхолдеры шаблона:** `{{char}}` (имя персонажа), `{{user}}` (ваше имя), `{{chat}}` (ID чата).
- **Умная нумерация:** автоматически добавляет числа (2, 3, 4...), если уже существуют дубликаты имени.
- **Примечание:** нельзя использовать одновременно с Manual Lorebook Mode.

### **Ручной режим лорбука**

- **Как это работает:** позволяет выбирать другой лорбук для воспоминаний отдельно для каждого чата, игнорируя основной лорбук, привязанный к чату.
- **Лучше всего для:** опытных пользователей, которые хотят направлять воспоминания в конкретный отдельный лорбук.
- **Как использовать:**
  1. Включите `Enable Manual Lorebook Mode` в настройках расширения.
  2. При первом создании воспоминания в чате появится запрос на выбор лорбука.
  3. Этот выбор сохраняется для конкретного чата, пока вы его не очистите или не вернетесь в Automatic Mode.
- **Примечание:** нельзя использовать одновременно с Auto-Create Lorebook Mode.

### 🎡 Трекеры и Side Prompts

![Где найти трекеры и Side Prompts](https://github.com/aikohanasaki/imagehost/blob/main/STMemoryBooks/sp.png)


> 📘 Для Side Prompts есть отдельное руководство: [Side Prompts Guide](side-prompts-ru.md). Используйте его для Sets, макросов, примеров и устранения неполадок.
> 🎡 Нужен точный путь по интерфейсу? См. [Scribe-инструкцию по включению Side Prompts](https://scribehow.com/viewer/How_to_Enable_Side_Prompts_in_Memory_Books__fif494uSSjCmxE2ZCmRGxQ).

Side Prompts — это отдельные запуски промтов STMB для поддержания текущего состояния чата. Используйте их для трекеров и вспомогательных заметок, которые не должны раздувать обычный ответ персонажа, например:

- 💰 Инвентарь и ресурсы («Какие предметы есть у пользователя?»)
- ❤️ Статус отношений («Что X чувствует к Y?»)
- 📊 Характеристики персонажа («Текущее здоровье, навыки, репутация»)
- 🎯 Прогресс квеста («Какие цели активны?»)
- 🌍 Состояние мира («Что изменилось в сеттинге?»)

#### **Доступ:** в настройках Memory Books нажмите `🎡 Trackers & Side Prompts`.

#### **Функции:**

- Просмотр, создание, дублирование, редактирование, удаление, экспорт и импорт Side Prompts.
- Запуск Side Prompts вручную, после создания воспоминания или как части Side Prompt Set.
- Использование стандартных макросов SillyTavern, например `{{user}}` и `{{char}}`.
- Использование runtime-макросов вроде `{{npc name}}`, когда промту нужно значение, передаваемое во время запуска.
- Сохранение вывода Side Prompt как отдельных side-prompt записей в лорбуке памяти.

#### **Советы по использованию:**

- При создании нового промта копируйте встроенные варианты.
- Side Prompts не обязаны возвращать JSON. Обычный текст или Markdown подходят.
- Side Prompts обычно обновляются/перезаписываются; воспоминания сохраняются последовательно.
- Ручной синтаксис: `/sideprompt "Name" {{macro}}="value" [X-Y]`.
- Используйте Side Prompt Sets, когда чату нужен упорядоченный набор трекеров.
- Выбранный after-memory Side Prompt Set заменяет для этого чата отдельно включенные after-memory Side Prompts.
- Дополнительная библиотека шаблонов Side Prompt: [JSON-файл](../resources/SidePromptTemplateLibrary.json). Просто импортируйте его для использования.

### 🧹 Сжатие

![Нажмите здесь, чтобы открыть меню сжатия](https://github.com/aikohanasaki/imagehost/blob/main/STMemoryBooks/compaction.png)


Сжатие — это процесс проверки, который помогает сделать записи лорбука, которыми управляет STMB, более эффективными по токенам. STMB просит ИИ переписать одну существующую запись, а затем показывает исходный текст и сжатый черновик до замены.

Открыть его можно из основного окна Memory Books кнопкой **📝 Сжатие**. Длинные записи-фрагменты также могут предлагать кнопку **Сжать запись** прямо в процессе работы с фрагментами.

#### Подходящие записи

Сжатие показывает подходящие записи из выбранной Книги Памяти:

- записи-фрагменты с пометкой `[STMB Clip]`
- записи Side Prompt
- записи памяти STMB, отмеченные Memory Books

Обычные записи лорбука, которыми STMB не управляет, не показываются.

#### Как это работает

1. Откройте Memory Books и нажмите **📝 Сжатие**.
2. Выберите **Книгу Памяти**. Если у текущего чата уже есть действительная Книга Памяти, STMB выберет её заранее; иначе выберите нужную в поисковом выпадающем списке.
3. Выберите **Профиль сжатия**. Он определяет, какое подключение/модель ИИ будет использоваться для запроса сжатия.
4. При необходимости нажмите **Редактировать промпт сжатия**, если хотите изменить инструкции, отправляемые ИИ.
5. Нажмите **Сжать запись** рядом с записью, которую хотите переписать.
6. Сравните **Исходное содержимое** и **Сжатый черновик**. STMB показывает примерную оценку токенов для обоих вариантов.
7. При необходимости отредактируйте черновик, затем выберите **Заменить сжатой версией**, **Копировать сжатый черновик** или **Отмена**.

STMB **не** заменяет оригинал автоматически. Запись лорбука меняется только после нажатия **Заменить сжатой версией**.

#### Промпт сжатия

Промпт сжатия можно редактировать. Встроенный промпт просит ИИ сохранять важные факты, имена, местоимения, макросы, служебные заголовки и конечные маркеры, одновременно убирая повторы и малополезные формулировки.

Поддерживаемые плейсхолдеры промпта:

- `{{ENTRY_CONTENT}}` — текущее содержимое записи лорбука. Этот плейсхолдер обязателен.
- `{{ENTRY_KIND}}` — тип записи, например Clip, SidePrompt или Memory.
- `{{ENTRY_TITLE}}` — название записи лорбука.

Используйте **Сбросить к значению по умолчанию** в редакторе промпта, если хотите вернуть встроенный Промпт сжатия.

#### Лучше всего подходит для

- длинных записей-фрагментов
- записей-трекеров Side Prompt, где накопились повторяющиеся заметки
- полезных, но слишком многословных записей памяти STMB
- всегда активных записей, которые начинают зря расходовать контекст

#### Не предназначено для

- добавления новых фактов
- суммаризации сырого чата
- создания новых воспоминаний
- переписывания обычных записей лорбука, которыми STMB не управляет

### 🧠 Интеграция Regex для продвинутой настройки

![Настройка regex](https://github.com/aikohanasaki/imagehost/blob/main/STMemoryBooks/regex.png)


- **Полный контроль над обработкой текста:** Memory Books интегрируется с расширением SillyTavern **Regex**, позволяя применять текстовые преобразования на двух ключевых этапах:
  1. **Генерация промта:** автоматически изменять промты, отправляемые ИИ, создавая regex-скрипты с размещением **User Input**.
  2. **Парсинг ответа:** очищать, переформатировать или стандартизировать сырой ответ ИИ перед сохранением, используя размещение **AI Output**.
- **Поддержка множественного выбора:** можно выбрать несколько скриптов для исходящей и входящей обработки.
- **Как это работает:** включите `Use regex (advanced)` в STMB, нажмите `📐 Configure regex…` и выберите, какие скрипты STMB должен запускать перед отправкой ИИ и перед парсингом/сохранением ответа.
- **Важно:** выбором Regex управляет STMB. Скрипты, выбранные там, будут запускаться **даже если они сейчас отключены в самом расширении Regex**.

## 🏷️ Форматирование заголовков

![Формат заголовка](https://github.com/aikohanasaki/imagehost/blob/main/STMemoryBooks/titleformat.png)
![Форматы заголовков](https://github.com/aikohanasaki/imagehost/blob/main/STMemoryBooks/titleformats.png)


Настраивайте заголовки записей лорбука с помощью мощной системы шаблонов.

- **Плейсхолдеры:**
  - `{{title}}` — заголовок, сгенерированный ИИ, например «Судьбоносная встреча».
  - `{{scene}}` — диапазон сообщений, например `Scene 15-23`.
  - `{{char}}` — имя персонажа.
  - `{{user}}` — ваше имя пользователя.
  - `{{messages}}` — количество сообщений в сцене.
  - `{{profile}}` — имя профиля, использованного для генерации.
  - плейсхолдеры текущей даты/времени в разных форматах, например `August 13, 2025` для даты и `11:08 PM` для времени.
- **Автонумерация:** используйте `[0]`, `[00]`, `(0)`, `{0}`, `#0`, а также обернутые формы вроде `#[000]`, `([000])`, `{[000]}` для последовательной нумерации с ведущими нулями.
- **Пользовательские форматы:** можно создавать собственные форматы. Начиная с v4.5.1, в заголовках разрешены все печатные Unicode-символы, включая emoji, CJK, буквы с диакритикой, символы и т. п. Блокируются только управляющие Unicode-символы.

## 🧾 Опциональная очередь задач (требуется Chat Top Bar)

![Очередь задач ST Memory Books](https://github.com/aikohanasaki/imagehost/blob/main/STMemoryBooks/queue.png)


Очередь задач необязательна, но это мощная функция. Она не нужна для обычного использования Memory Books.

Если установить и включить **Chat Top Bar** / **Chat Top Info Bar**, STMB добавит кнопку **Задачи Книг Памяти** на верхнюю панель чата. Она открывает боковую панель очереди, где можно видеть активные, завершённые, неудачные, отменённые задачи Memory Books, а также задачи, требующие проверки.

Это особенно полезно, когда вы:

- создаёте воспоминания из длинных сцен
- запускаете консолидацию
- запускаете Side Prompts после создания воспоминаний
- работаете в длинных чатах и хотите яснее видеть прогресс и обработку проверок

Очередь может показывать статус задач, отменять активные задачи, повторять неудачные и скрывать завершённые. Если задаче в очереди нужна проверка пользователя, STMB может пометить её как **Требует проверки** вместо того, чтобы незаметно перезаписать что-то небезопасное.

Если Chat Top Bar не установлен или не включён, STMB продолжит работать обычным образом. Просто интерфейс очереди задач будет недоступен.

# FAQ

### Должен ли я создавать отдельный лорбук для воспоминаний или можно использовать тот же лорбук, что и для других вещей?

Я рекомендую делать лорбук памяти отдельной книгой. Так проще организовывать воспоминания отдельно от других записей. Например, его проще добавить в групповой чат, использовать в другом чате или назначить отдельный бюджет лорбука через STLO.

### Нужны ли мне Vectors?

Можно использовать, но это не обязательно. Если вы не используете расширение vectors (я не использую), Memory Books работает через ключевые слова. Это автоматизировано, поэтому вам не нужно думать о том, какие ключевые слова использовать.

### Стоит ли использовать Delay until recursion, если Memory Books — единственный лорбук?

Нет. Если нет других World Info или лорбуков, `Delay until recursion` может помешать первому циклу сработать, и ничего не активируется. Если Memory Books — единственный лорбук, отключите `Delay until recursion` или добавьте хотя бы один дополнительный World Info/лорбук.

### Почему ИИ не видит мои записи?

Сначала проверьте, что записи действительно отправляются. Для этого удобно использовать [WorldInfo-Info](https://github.com/aikohanasaki/SillyTavern-WorldInfoInfo).

Если записи срабатывают и отправляются ИИ, вероятно, придется резко напомнить ИИ об этом в OOC. Например: `[OOC: WHY are you not using the information you were given? Specifically: (whatever it was)]`

# Troubleshooting

![Предупреждение о пересечении сцен](https://github.com/aikohanasaki/imagehost/blob/main/STMemoryBooks/overlap.png)
![Включить пересечение сцен](https://github.com/aikohanasaki/imagehost/blob/main/STMemoryBooks/overlap2.png)


- **Я не могу найти Memory Books в меню Extensions!**  
  Настройки находятся в меню Extensions (волшебная палочка 🪄 слева от поля ввода). Ищите `Memory Books`.

![Расположение настроек STMB](https://github.com/aikohanasaki/imagehost/blob/main/STMemoryBooks/menu.png)


- **Лорбук недоступен или не выбран:**
  - В Manual Mode выберите лорбук при запросе.
  - В Automatic Mode привяжите лорбук к вашему чату.
  - Либо включите `Auto-create lorebook if none exists` для автоматического создания.

- **Lorebook Validation Error:**
  - Вероятно, вы удалили ранее привязанный лорбук. Просто привяжите новый лорбук; он может быть пустым.

- **Сцена не выбрана:**
  - Отметьте и начало (►), и конец (◄).

- **Сцена перекрывается с существующим воспоминанием:**
  - Выберите другой диапазон или включите `Allow scene overlap` в настройках.


- **ИИ не смог создать валидное воспоминание:**
  - Используйте модель, которая поддерживает JSON output.
  - Проверьте промт и настройки модели.

- **Превышен порог предупреждения о токенах:**
  - Используйте меньшую сцену или увеличьте порог.

- **Нет кнопок-шевронов:**
  - Дождитесь загрузки расширения или обновите страницу.

- **Данные персонажа недоступны:**
  - Дождитесь полной загрузки чата/группы.

## 📝 Политика символов (v4.5.1+)

- **Разрешено в заголовках:** все печатные Unicode-символы, включая буквы с диакритикой, emoji, CJK и символы.
- **Блокируется:** только управляющие Unicode-символы (U+0000–U+001F, U+007F–U+009F); они удаляются автоматически.

См. [Character Policy Details](../charset.md) для примеров и заметок по миграции.

## 👨‍💻 Для разработчиков

### Сборка расширения

Это расширение использует Bun для сборки. Процесс сборки минифицирует и упаковывает исходные файлы.

```sh
# Сборка расширения
bun run build
```

### Git hooks

Проект включает pre-commit hook, который автоматически собирает расширение и добавляет build artifacts в коммиты. Это помогает держать собранные файлы синхронизированными с исходниками.

**Чтобы установить git hook:**

```sh
bun run install-hooks
```

Hook выполняет следующее:

- запускает `bun run build` перед каждым коммитом;
- добавляет build artifacts в коммит;
- отменяет коммит, если сборка не удалась.
