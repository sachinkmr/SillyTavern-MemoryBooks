# 📕 Memory Books (Расширение для SillyTavern)

Расширение нового поколения для SillyTavern, предназначенное для автоматического, структурированного и надежного создания памяти. Отмечайте сцены в чате, генерируйте JSON-сводки с помощью ИИ и сохраняйте их как записи в ваших лорбуках (книгах знаний). Поддерживает групповые чаты, расширенное управление профилями, side prompts/трекеры и многоуровневые сводки.

### ❓ Словарь
- Scene (Сцена) → Memory (Воспоминание)
- Many Memories (Много воспоминаний) → Summary / Consolidation (Сводка / Консолидация)
- Always-On (Всегда включено) → Side Prompt (Tracker) / Побочный промт (Трекер)

## 📋 Предварительные требования (Prerequisites)

- **SillyTavern:** 1.14.0+ (рекомендуется последняя версия).
- **Выбор сцены:** Должны быть установлены маркеры начала и конца (начало < конец).
- **Поддержка Chat Completion:** Полная поддержка OpenAI, Claude, Anthropic, OpenRouter или других API завершения чата.
- **Поддержка Text Completion:** API завершения текста (Kobold, TextGen и т.д.) поддерживаются при подключении через конечную точку API Chat Completion (OpenAI-совместимую). Я рекомендую настроить подключение Chat Completion API в соответствии с советами для KoboldCpp ниже (измените при необходимости, если используете Ollama или другое ПО). После этого настройте профиль STMB и используйте Custom (рекомендуется) или полную ручную конфигурацию (только если Custom не работает или у вас более одного пользовательского подключения).
**ПРИМЕЧАНИЕ**: Обратите внимание, что если вы используете Text Completion, вы должны...

### Советы по использованию 📕 ST Memory Books с KoboldCpp
Настройте это в ST (вы можете переключиться обратно на Text Completion ПОСЛЕ того, как настроите STMB):
- API: Chat Completion
- Источник: Custom chat completion source (Пользовательский)
- Эндпоинт: `http://localhost:5001/v1` (вы также можете использовать `127.0.0.1:5000/v1`)
- Введите что угодно в "custom API key" (не имеет значения, но ST требует наличие ключа).
- Model ID должен быть `koboldcpp/modelname` (не указывайте .gguf в имени модели!).
- Скачайте пресет для chat completion и импортируйте его (подойдет любой), просто чтобы у вас БЫЛ пресет. Это позволит избежать ошибок "not supported".
- Измените максимальную длину ответа в пресете chat completion так, чтобы она была не менее 2048; рекомендуется 4096. (Меньшее значение может привести к обрезанию ответа).

### Советы по использованию 📕 ST Memory Books с Llama.cpp
Так же, как и с Kobold, настройте следующее как _Chat Completion API_ в ST (вы можете переключиться обратно после проверки работы STMB):
- Создайте новый профиль подключения для Chat Completion API.
- Источник завершения (Completion Source): `Custom (Open-AI Compatible)`.
- URL эндпоинта: `http://host.docker.internal:8080/v1`, если ST запущен в docker, иначе `http://localhost:8080/v1`.
- Custom API key: введите что угодно (ST требует ключ).
- Model ID: `llama2-7b-chat.gguf` (или ваша модель, неважно, если в llama.cpp запущена только одна).
- Постобработка промта (Prompt post-processing): нет.

Для запуска Llama.cpp я рекомендую поместить что-то подобное в shell-скрипт или bat-файл для упрощения запуска:
```sh
llama-server -m <путь-к-модели> -c <размер-контекста> --port 8080

```

## 💡 Рекомендуемые глобальные настройки активации World Info/Lorebook

* **Match Whole Words (Только целые слова):** оставить выключенным (false).
* **Scan Depth (Глубина сканирования):** чем выше, тем лучше (у меня установлено 8).
* **Max Recursion Steps (Макс. шагов рекурсии):** 2 (общая рекомендация, не обязательно).
* **Context % (% контекста):** 80% (исходя из окна контекста в 100,000 токенов) — предполагается, что у вас нет супер-тяжелой истории чата или ботов.

## 🚀 Начало работы

### 1. **Установка и Загрузка**

* Загрузите SillyTavern и выберите персонажа или групповой чат.
* Дождитесь появления кнопок-шевронов (► ◄) на сообщениях чата (может занять до 10 секунд).

### 2. **Отметьте сцену**

* Нажмите ► на первом сообщении вашей сцены.
* Нажмите ◄ на последнем сообщении.

### 3. **Создайте воспоминание**

* Откройте меню Расширений (волшебная палочка 🪄) и нажмите "Memory Books", или используйте слэш-команду `/creatememory`.
* Подтвердите настройки (профиль, контекст, API/модель), если появится запрос.
* Дождитесь генерации ИИ и автоматического создания записи в лорбуке.

---

## 🆕 Быстрые слэш-команды

* `/creatememory` использует существующие маркеры начала/конца (шевроны) для создания воспоминания.
* `/scenememory x-y` создаст воспоминание, начинающееся с сообщения x и заканчивающееся сообщением y.
* `/nextmemory` создаст воспоминание из всех сообщений после последнего воспоминания.

## 👥 Поддержка групповых чатов

* Все функции работают с групповыми чатами.
* Маркеры сцен, создание памяти и интеграция с лорбуком хранятся в метаданных группы.
* Специальная настройка не требуется — просто выберите групповой чат и используйте как обычно.

## 🧭 Режимы работы

### **Автоматический режим (По умолчанию)**

* **Как это работает:** Автоматически использует лорбук, привязанный к вашему текущему чату.
* **Для чего лучше:** Простота и скорость. Большинству пользователей стоит начать с этого.
* **Как использовать:** Убедитесь, что лорбук выбран в выпадающем меню "Chat Lorebooks" для вашего персонажа или группового чата.

### **Режим авто-создания лорбука** ⭐ *Новое в v4.2.0*

* **Как это работает:** Автоматически создает и привязывает новый лорбук, если его нет, используя ваш шаблон именования.
* **Для чего лучше:** Для новых пользователей и быстрой настройки. Идеально для создания лорбука в один клик.
* **Как использовать:**
1. Включите "Автоматически создавать лорбук, если он не существует" в настройках расширения.
2. Настройте шаблон именования (по умолчанию: "LTM - {{char}} - {{chat}}").
3. Когда вы создаете воспоминание без привязанного лорбука, он автоматически создается и привязывается.


* **Заполнители шаблона:** {{char}} (имя персонажа), {{user}} (ваше имя), {{chat}} (ID чата).
* **Умная нумерация:** Автоматически добавляет цифры (2, 3, 4...), если существуют дубликаты имен.
* **Примечание:** Не может использоваться одновременно с Ручным режимом лорбука.

### **Ручной режим лорбука**

* **Как это работает:** Позволяет выбирать другой лорбук для воспоминаний отдельно для каждого чата, игнорируя основной лорбук, привязанный к чату.
* **Для чего лучше:** Для опытных пользователей, которые хотят направлять воспоминания в конкретный, отдельный лорбук.
* **Как использовать:**
1. Включите "Включить ручной режим Лора" в настройках расширения.
2. При первом создании воспоминания в чате вам будет предложено выбрать лорбук.
3. Этот выбор сохраняется для конкретного чата, пока вы не сбросите его или не переключитесь обратно в Автоматический режим.


* **Примечание:** Не может использоваться одновременно с Режимом авто-создания лорбука.

## 🧩 Типы памяти: Сцены против Сводок

📕 Memory Books поддерживает **воспоминания о сценах** и **многоуровневые сводки**, каждый из которых предназначен для разных видов непрерывности.

### 🎬 Scene Memories / Воспоминания о сценах (По умолчанию)

Воспоминания о сценах фиксируют то, **что произошло** в определенном диапазоне сообщений.

* Основаны на явном выборе сцены (► ◄).
* Идеальны для запоминания моментов.
* Сохраняют диалоги, действия и непосредственные результаты.
* Лучше всего использовать часто.

Это стандартный и наиболее часто используемый тип памяти.

---

### 🌈 Сводки

Сводки фиксируют, **что изменилось со временем**, и опираются на существующие воспоминания STMB.

Вместо того чтобы просто суммировать одну сцену, сводки фокусируются на:

* Развитии персонажа и сдвигах в отношениях
* Долгосрочных целях, напряженности и разрешениях конфликтов
* Эмоциональной траектории и направлении повествования
* Постоянных изменениях состояния, которые должны оставаться стабильными

Первый уровень консолидации - **Arc**, он строится из воспоминаний о сценах. Для более длинных историй есть и более высокие уровни:

* Arc
* Chapter
* Book
* Legend
* Series
* Epic

> 💡 Думайте об этих сводках как о *кратких обзорах*, а не как о логах сцен.

#### Когда использовать консолидацию

* После серьезного сдвига в отношениях
* В конце главы истории или арки
* Когда меняются мотивация, доверие или динамика власти
* Перед началом новой фазы истории

#### Как это работает

* Сводки генерируются из существующих воспоминаний STMB, а не напрямую из необработанного чата
* Инструмент **Объединить воспоминания** позволяет выбрать целевой уровень и источники
* STMB может отслеживать выбранные уровни и показывать подтверждение Да/Позже, когда достигнут сохраненный минимум
* STMB может отключать исходные записи после консолидации, если вы хотите, чтобы верхний уровень взял на себя роль
* Неудачные ответы ИИ можно проверить и исправить в интерфейсе перед повторной попыткой сохранения

Это дает вам:

* меньшее использование токенов
* лучшую непрерывность повествования в длинных чатах

## 📝 Генерация памяти

### **Только JSON на выходе**

Все промты и пресеты **должны** инструктировать ИИ возвращать только валидный JSON, например:

```json
{
  "title": "Название короткой сцены",
  "content": "Детальное резюме сцены...",
  "keywords": ["ключевоеслово1", "ключевоеслово2"]
}

```

**Никакой другой текст в ответе не допускается.**

### **Встроенные пресеты**

1. **Summary:** Детальные пошаговые (beat-by-beat) сводки.
2. **Summarize:** Заголовки Markdown для хронологии, битов, взаимодействий, результатов.
3. **Synopsis:** Всеобъемлющий, структурированный markdown.
4. **Sum Up:** Лаконичная сводка битов с хронологией.
5. **Minimal:** Сводка в 1-2 предложения.
6. **Northgate:** Литературный стиль сводки для творческого письма.
7. **Aelemar:** Фокусируется на сюжетных точках и воспоминаниях персонажей.
8. **Comprehensive:** Сводка в стиле synopsis с улучшенным извлечением ключевых слов.

### **Пользовательские промты**

* Создавайте свои собственные, но они **должны** возвращать валидный JSON, как указано выше.

## 📚 Интеграция с Лорбуком

* **Автоматическое создание записей:** Новые воспоминания сохраняются как записи со всеми метаданными.
* **Обнаружение по флагу:** Только записи с флагом `stmemorybooks` распознаются как воспоминания.
* **Авто-нумерация:** Последовательная нумерация с нулями в начале, поддерживаются различные форматы (`[000]`, `(000)`, `{000}`, `#000`).
* **Ручной/Автоматический порядок:** Настройки порядка вставки для каждого профиля.
* **Обновление редактора:** Опциональное автоматическое обновление редактора лорбука после добавления воспоминания.

> **Существующие воспоминания должны быть конвертированы!**
> Используйте [Конвертер Лорбуков](../resources/lorebookconverter.html) для добавления флага `stmemorybooks` и необходимых полей.

### 🎡 Трекеры и Боковые промты

Побочные промты могут использоваться как трекеры и будут создавать отдельные записи side prompt в вашем лорбуке памяти. Они позволяют отслеживать **текущее состояние**, а не только прошлые события. Например:

* 💰 Инвентарь и Ресурсы ("Какие предметы есть у пользователя?")
* ❤️ Статус отношений ("Что X чувствует к Y?")
* 📊 Характеристики персонажа ("Текущее здоровье, навыки, репутация")
* 🎯 Прогресс квеста ("Какие цели активны?")
* 🌍 Состояние мира ("Что изменилось в сеттинге?")

#### **Доступ:** Из настроек Memory Books нажмите “🎡 Трекеры и Боковые промты” (Менеджер побочных промтов).

#### **Функции:**

```
- Просмотр всех побочных промтов.
- Создание новых или дублирование промтов для экспериментов с разными стилями.
- Редактирование или удаление любого пресета (включая встроенные).
- Экспорт и импорт пресетов в виде JSON-файлов для бэкапа или обмена.
- Запуск вручную или автоматически, в зависимости от шаблона.
- Использовать стандартные ST-макросы вроде `{{user}}` и `{{char}}` в `Prompt` и `Response Format`.
- Использовать пользовательские runtime-макросы вроде `{{npc name}}`, которые передаются при запуске `/sideprompt`.

```

#### **Советы по использованию:**

```
- При создании нового промта вы можете скопировать встроенные для лучшей совместимости.
- Дополнительная библиотека шаблонов побочных промтов [файл JSON](../resources/SidePromptTemplateLibrary.json) — просто импортируйте для использования.
- Ручной синтаксис: `/sideprompt "Имя" {{macro}}="value" [X-Y]`.
- После выбора side prompt в автодополнении команд STMB предложит недостающие runtime-макросы.
- Side prompt с пользовательскими runtime-макросами доступны только для ручного запуска. STMB удаляет `On Interval` и `On After Memory` у таких шаблонов при сохранении/импорте и показывает предупреждение.

```

### 🧠 Интеграция Regex для продвинутой настройки

* **Полный контроль над обработкой текста**: Memory Books теперь интегрируется с расширением **Regex** (Регулярные выражения) для SillyTavern, позволяя применять мощные преобразования текста на двух ключевых этапах:
1. **Генерация промта**: Автоматическое изменение промтов, отправляемых ИИ, путем создания regex-скриптов, нацеленных на размещение **User Input**.
2. **Парсинг ответа**: Очистка, переформатирование или стандартизация необработанного ответа ИИ перед его сохранением путем нацеливания на размещение **AI Output**.


* **Поддержка множественного выбора**: Вы можете выбрать несколько regex-скриптов.
* **Как это работает**: Включите в STMB `Использовать регулярные выражения (расширенный)`, нажмите `📐 Настроить регулярные выражения…` и выберите, какие скрипты STMB должен запускать перед отправкой в ИИ и перед разбором/сохранением ответа.
* **Важно**: Выбором управляет STMB. Скрипты, выбранные там, выполняются **даже если они отключены** в самом расширении Regex.

# FAQ (ЧаВо)

### Я не могу найти Memory Books в меню Расширений!

Настройки находятся в меню Расширений (волшебная палочка 🪄 слева от поля ввода). Ищите "Memory Books".

### Почему ИИ не видит мои записи?

Сначала проверьте, что записи вообще отправляются. Для этого я использую [WorldInfo-Info](https://github.com/aikohanasaki/SillyTavern-WorldInfoInfo). Если они отправляются, но ИИ все равно их игнорирует, вероятно, нужно напомнить ему об этом более явно в OOC.

### Нужны ли мне векторы (Vectors)?

Запись 🔗 в "мировой информации" (world info) в интерфейсе ST называется "vectorized". Именно поэтому я использую слово "векторизованный". Если вы не используете расширение векторов (я не использую), это работает через ключевые слова. Все это автоматизировано, так что вам не нужно думать о том, какие ключевые слова использовать.

### Должен ли я создавать отдельный лорбук для воспоминаний или могу использовать тот же, что и для других вещей?

Я рекомендую, чтобы ваш лорбук памяти был отдельной книгой. Это облегчает организацию воспоминаний (в отличие от других записей). Например, добавление его в групповой чат, использование в другом чате или установка индивидуального бюджета лорбука (с использованием STLO).

### Стоит ли использовать 'Отложить до рекурсии', если Memory Books — единственный лорбук?

Нет. Если нет другой мировой информации или лорбуков, выбор 'Отложить до рекурсии' может предотвратить запуск первого цикла, из-за чего ничего не активируется. Если Memory Books — единственный лорбук, либо отключите 'Отложить до рекурсии', либо убедитесь, что настроена хотя бы одна дополнительная мировая информация/лорбук.

# Устранение неполадок (Troubleshooting)

* **Лорбук недоступен или не выбран:**
* В Ручном режиме выберите лорбук при запросе.
* В Автоматическом режиме привяжите лорбук к вашему чату.
* Или включите "Автоматически создавать лорбук, если он не существует" для автоматического создания.


* **Сцена не выбрана:**
* Отметьте как точку начала (►), так и точку конца (◄).

* **Связанный лорбук исчез или был удален:**
* Просто привяжите новый лорбук, даже пустой.


* **Сцена перекрывается с существующим воспоминанием:**
* Выберите другой диапазон или включите "Allow scene overlap" (Разрешить перекрытие сцен) в настройках.


* **ИИ не смог сгенерировать валидное воспоминание:**
* Используйте модель, которая поддерживает вывод JSON.
* Проверьте настройки промта и модели.


* **Превышен порог предупреждения о токенах:**
* Используйте меньшую сцену или увеличьте порог.


* **Отсутствуют кнопки-шевроны:**
* Подождите загрузки расширения или обновите страницу.


* **Данные персонажа недоступны:**
* Подождите полной загрузки чата/группы.
