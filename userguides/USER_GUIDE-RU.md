# 📕 ST Memory Books — Ваш ИИ-помощник для памяти в чате

**Превратите бесконечные переписки в организованные, доступные для поиска воспоминания!**

Нужно, чтобы бот помнил детали, но контекст чата уже слишком велик? Хотите автоматически отслеживать важные сюжетные моменты, не ведя заметки вручную? ST Memory Books делает именно это: следит за чатом, создаёт умные сводки и помогает не терять нить истории.

(Если вам нужны технические детали, загляните в [Как работает STMB](howSTMBworks-ru.md).)

## 🚀 Быстрый старт (Первое воспоминание за 5 минут)

**Впервые пользуетесь ST Memory Books?** Давайте быстро настроим первое автоматическое воспоминание.

### Шаг 1: Найдите расширение

- Найдите иконку волшебной палочки (🪄) рядом с полем ввода чата
- Нажмите на неё и выберите **"Memory Books"**
- Откроется панель управления ST Memory Books

### Шаг 2: Включите авто-режим

- В панели найдите **`Автоматически создавать сводки памяти`**
- Включите этот переключатель
- Поставьте **`Интервал Авто-Сводки`** примерно на **20-30 сообщений** для старта
- **`Буфер Авто-Сводки`** сначала лучше держать низким, например `0-2`
- Сначала создайте одну память вручную, чтобы чат был подготовлен к авто-режиму
- Готово

### Шаг 3: Просто продолжайте чат

- Общайтесь как обычно
- После накопления новых сообщений ST Memory Books автоматически:
  - возьмёт новые сообщения после последней обработанной точки
  - попросит ИИ написать сводку
  - сохранит её в коллекцию памяти
  - покажет уведомление после завершения

**Готово.** Теперь памятью можно управлять автоматически, без ручной рутины после каждой сцены.

## 💡 Что ST Memory Books делает на практике

Думайте о ST Memory Books как о **личном ИИ-библиотекаре** для ваших чатов.

### 🤖 Автоматические сводки

*"Я не хочу об этом думать, пусть всё работает само"*

- Следит за чатом в фоне
- Автоматически создаёт воспоминания через заданные интервалы
- Отлично подходит для длинных ролевых игр, творчества и непрерывных сюжетов

### ✋ Ручное создание памяти

*"Я хочу сам решать, что сохранять"*

- Позволяет отмечать важные сцены кнопками-стрелками (► ◄)
- Даёт создавать память по запросу для особых моментов
- Удобно для ключевых поворотов сюжета и важных сцен

### 📊 Боковые промпты и умные трекеры

*"Я хочу отслеживать отношения, сюжетные линии или статусы"*

- Используют переиспользуемые промпты рядом с обычной памятью
- Поддерживают библиотеку шаблонов
- Могут отслеживать почти любые состояния
- Подходят для таблиц, отношений, текущих целей, заметок по миру и т.п.

### 📚 Коллекции памяти

*Там, где всё это живёт*

- Автоматически организуются и доступны для поиска
- Работают через систему lorebook в SillyTavern
- Могут использоваться ИИ в новых сообщениях как опора на прошлое

## 🎯 Выберите свой стиль

<details>
<summary><strong>🔄 «Настроил и забыл» (лучше всего для новичков)</strong></summary>

**Подходит, если:** вы хотите, чтобы всё работало само.

**Как это работает:**
1. Включите `Автоматически создавать сводки памяти`
2. Подберите удобный `Интервал Авто-Сводки`
3. При желании задайте небольшой `Буфер Авто-Сводки`
4. После одной ручной памяти просто продолжайте чат

**Что это даёт:**
- почти никакой ручной работы
- стабильное создание памяти
- меньше шансов потерять важные сюжетные моменты
- работает и в одиночных, и в групповых чатах

**Совет:** начните с `30`, а потом скорректируйте интервал под свой темп чата.

</details>

<details>
<summary><strong>✋ «Ручной контроль» (если хотите выбирать сцены сами)</strong></summary>

**Подходит, если:** вы хотите сами решать, что станет памятью.

**Как это работает:**
1. Найдите кнопки (► ◄) у сообщений чата
2. Нажмите ► на первом сообщении важной сцены
3. Нажмите ◄ на последнем сообщении сцены
4. Откройте Memory Books и нажмите **`Создать память`**

**Что это даёт:**
- полный контроль над содержимым памяти
- удобство для важных сюжетных сцен
- точную работу с границами эпизода

**Совет:** кнопки могут появляться с небольшой задержкой после загрузки чата. Если их нет сразу, подождите или обновите страницу.

</details>

<details>
<summary><strong>⚡ «Продвинутый пользователь» (slash-команды)</strong></summary>

**Подходит, если:** вы хотите работать через команды и быстрые сценарии.

**Основные команды:**
- `/scenememory 10-25` — создать память по сообщениям 10-25
- `/creatememory` — создать память по уже отмеченной сцене
- `/nextmemory` — свести всё от последней памяти до текущего момента
- `/sideprompt "Relationship Tracker" {{macro}}="value" [X-Y]` — вручную запустить side prompt с необязательным диапазоном
- `/sideprompt-on "Name"` или `/sideprompt-off "Name"` — включить или выключить side prompt
- `/stmb-set-highest <N|none>` — вручную скорректировать базовую точку для авто-сводок

**Что это даёт:**
- быстрый рабочий процесс
- пакетные действия
- удобство для собственных сценариев использования

</details>

## 🙈 Экономия токенов: скрытие и показ сообщений

Один из самых простых способов сократить захламление и сэкономить токены в длинных чатах — скрывать сообщения после того, как вы уже превратили их в память.

### Что значит «скрыть»?

Скрытие **не удаляет** сообщения. Оно просто убирает их из прямой подачи в ИИ. Сообщения чата по-прежнему существуют, а сами воспоминания остаются в lorebook, так что информация не теряется.

### Зачем это использовать?

Скрытие и повторный показ полезны, когда:

- чат уже стал очень длинным
- для этих сообщений память уже создана
- вы хотите уменьшить «шум» в истории

### Автоскрытие после создания памяти

STMB может автоматически скрывать сообщения после создания памяти. Есть варианты:

- **`Не скрывать автоматически`** — ничего не скрывать
- **`Автоматически скрывать все сообщения до последней памяти`** — скрывать всё, что уже покрыто памятью
- **`Автоматически скрывать только сообщения последней памяти`** — скрывать только последний обработанный диапазон

Также можно указать, сколько последних сообщений оставить видимыми через **`Сообщения, которые нужно оставить видимыми`**.

### Показывать скрытое перед генерацией памяти

Опция **`Показывать скрытые сообщения для генерации памяти (выполняет /unhide X-Y)`** заставляет STMB временно выполнять `/unhide X-Y` для выбранного диапазона перед созданием памяти. Это особенно полезно, если вы часто переделываете память по уже скрытым сообщениям.

### Хорошая стартовая настройка

Базовый и безопасный набор:

- использовать **`Автоматически скрывать все сообщения до последней памяти`**
- оставлять видимыми **2** последних сообщения
- включить **`Показывать скрытые сообщения для генерации памяти (выполняет /unhide X-Y)`**

## 🌈 Консолидация сводок

Консолидация помогает держать длинные истории под контролем, сжимая старые STMB-воспоминания в более крупные обзорные записи.

### Что такое консолидация?

Вместо бесконечного накопления сценовых воспоминаний STMB умеет брать уже существующие памяти или сводки и превращать их в более компактный обзор. Первый уровень — **Arc**, а дальше есть и более высокие уровни:

- Arc
- Chapter
- Book
- Legend
- Series
- Epic

### Когда это полезно?

Консолидация нужна, когда:

- список памяти становится слишком длинным
- старая часть истории уже не требует покадровой детализации
- вы хотите снизить расход токенов, не потеряв непрерывность сюжета
- вам нужны более крупные обзорные сводки

### Запускается ли это автоматически?

Нет. Консолидация всё равно требует подтверждения.

- Вы всегда можете открыть **`Консолидировать воспоминания`** вручную из основного окна STMB
- Можно включить **`Запрашивать консолидацию, когда уровень готов`**
- Когда выбранный уровень достигает сохранённого минимума подходящих источников, STMB показывает подтверждение yes/later
- Нажатие **Yes** только открывает окно консолидации с нужным уровнем, а не запускает её молча в фоне

### Что именно попадает в консолидацию?

Консолидация работает с **обычными STMB-памятями и STMB-сводками**.

Это значит:

- обычные памяти можно объединять в более высокие сводки
- более высокие сводки можно затем объединять ещё выше

Side prompts — это другое.

**Side prompts — это трекерные записи**, а не записи типа memory/summary. Они нужны для поддержания текущего состояния:

- отношений
- целей
- счётчиков
- состояния мира
- текущих трекеров сюжета

Проще говоря:

- **Memory** = «что произошло в этой сцене?»
- **Side Prompt** = «каково текущее состояние этой штуки?»
- **Consolidation** = «свести много воспоминаний в более крупный обзор»

### Почему это важно?

Пользователи часто ожидают, что трекеры автоматически попадут в Arc или Chapter summary.

Они **не попадают**.

Если вы хотите, чтобы что-то участвовало в консолидации, это должно существовать как обычная память или сводка STMB, а не только как side prompt.

### Как этим пользоваться?

1. Нажмите **`Консолидировать воспоминания`** в основном окне STMB
2. Выберите целевой уровень summary
3. Отметьте исходные записи
4. При желании отключите исходные записи после создания новой сводки
5. Нажмите **`Run`**

Если ИИ вернул плохой ответ для консолидации, STMB может показать режим просмотра и исправления перед повторной попыткой сохранения.

## 🎨 Трекеры, боковые промпты и шаблоны (расширенная функция)

**Боковые промпты** — это фоновые трекеры, которые помогают поддерживать актуальную информацию по ходу истории. Они работают рядом с обычной памятью и обновляют отдельные записи lorebook со временем.

### 🚀 Быстрый старт с шаблонами

1. Откройте настройки Memory Books
2. Нажмите **`Боковые промпты`**
3. Откройте библиотеку шаблонов и выберите подходящие:
   - трекер развития персонажа
   - трекер отношений
   - трекер сюжетных линий
   - трекер настроения и атмосферы
   - заметки по миру
4. Включите нужные шаблоны
5. Если у шаблона есть автоматические триггеры, STMB будет обновлять эту запись рядом с обычной памятью

[Пошаговый Scribe по включению автоматических side prompts](https://scribehow.com/viewer/How_to_Enable_Side_Prompts_in_Memory_Books__fif494uSSjCmxE2ZCmRGxQ)

### ⚙️ Как работают боковые промпты

- **Фоновые трекеры:** тихо обновляют данные со временем
- **Ненавязчивость:** не заменяют основные настройки ИИ или персонажа
- **Отдельное состояние для каждого чата:** в разных чатах могут быть разные трекеры
- **Шаблоны:** можно использовать встроенные или создать свои
- **Автоматический и ручной режим:** обычные шаблоны могут запускаться автоматически, а шаблоны с кастомными runtime-макросами — только вручную
- **Поддержка макросов:** `Prompt`, `Response Format`, `Title` и поля ключевых слов могут использовать стандартные макросы ST вроде `{{user}}` и `{{char}}`
- **Runtime-макросы:** нестандартные `{{...}}` превращаются в обязательные аргументы команды, например `{{npc name}}="Jane Doe"`
- **Можно без JSON:** side prompts не обязаны возвращать JSON
- **Поведение обновления:** side prompts обычно обновляют собственную запись, а не создают каждый раз новую память по порядку

### 🛠️ Управление боковыми промптами

- **Менеджер боковых промптов:** создание, редактирование, дублирование и организация шаблонов
- **Enable / Disable:** включение и выключение трекеров
- **Import / Export:** резервные копии и обмен шаблонами
- **Просмотр статуса:** можно увидеть, какие side prompts активны и когда они запускаются
- **Проверки безопасности:** если в шаблоне есть кастомные runtime-макросы, STMB убирает автоматические триггеры при сохранении/импорте и показывает предупреждение

### 💡 Примеры шаблонов

- Библиотека side prompt шаблонов: [SidePromptTemplateLibrary.json](/resources/SidePromptTemplateLibrary.json)

Примеры идей:

- отслеживать важные диалоги и взаимодействия
- держать актуальным статус текущего квеста
- записывать новые детали мира
- отслеживать отношения между персонажами

### 🔧 Как создать свой Side Prompt

1. Откройте Менеджер боковых промптов
2. Нажмите **`Создать новый`**
3. Напишите короткую и понятную инструкцию
4. При желании добавьте стандартные ST-макросы вроде `{{user}}` и `{{char}}`
5. Если вы добавляете кастомные runtime-макросы вроде `{{location name}}`, запускайте такой шаблон вручную через `/sideprompt`
6. Сохраните и включите шаблон
7. Дальше трекер будет обновляться автоматически, если это допускает его тип, либо вручную по команде

### 💬 Практический совет

Side prompts лучше работают, когда они **узкие и конкретные**.

Вместо «следить вообще за всем» лучше писать что-то вроде «следить за романтическим напряжением между главными героями».

### ⌨️ Ручной синтаксис `/sideprompt`

Используйте:

`/sideprompt "Name" {{macro}}="value" [X-Y]`

Примеры:

- `/sideprompt "Status" 10-20`
- `/sideprompt "NPC Directory" {{npc name}}="Jane Doe" 40-50`
- `/sideprompt "Location Notes" {{place name}}="Black Harbor" 100-120`

Важно:

- имя side prompt должно быть в кавычках
- значения runtime-макросов тоже должны быть в кавычках
- автодополнение slash-команд подскажет обязательные runtime-макросы после выбора шаблона
- если шаблон содержит кастомные runtime-макросы, STMB принудительно оставляет его ручным
- `X-Y` не обязателен; если его нет, STMB берёт сообщения с момента последнего обновления этого side prompt

## 🧠 Продвинутый контроль текста через Regex

STMB может запускать выбранные Regex-скрипты **перед отправкой текста в ИИ** и **перед сохранением ответа**.

Это полезно, если вы хотите:

- чистить типичный мусор из ответов ИИ
- нормализовать имена и термины перед генерацией
- преобразовывать текст перед разбором или предпросмотром

### Как это работает сейчас

1. Создайте нужные Regex-скрипты в расширении **Regex** в SillyTavern
2. В STMB включите **`Использовать regex (для опытных)`**
3. Нажмите **`📐 Настроить regex…`**
4. Выберите, какие скрипты STMB должен запускать:
   - перед отправкой текста в ИИ
   - перед добавлением ответа в lorebook

### Что важно понимать

- выбор Regex для STMB управляется **внутри самого STMB**, а не только флагом enabled/disabled в Regex extension
- скрипт, выбранный в STMB, может сработать, даже если он выключен в Regex extension
- STMB поддерживает мультивыбор и для outgoing, и для incoming обработки

### Быстрый пример

Если модель постоянно добавляет `(OOC: I hope this summary is helpful!)`, можно:

1. создать Regex-скрипт, который удаляет этот текст
2. включить **`Использовать regex (для опытных)`** в STMB
3. открыть **`📐 Настроить regex…`**
4. добавить этот скрипт в список **incoming**

Тогда STMB очистит ответ ещё до предпросмотра и сохранения.

## ⚙️ Какие настройки важны в первую очередь

Этот гайд **не является полной справкой по настройкам**. Полный список смотрите в [readme-RU.md](readme-RU.md).

Но большинству пользователей в первую очередь полезно понимать вот это:

- **`Текущие настройки SillyTavern`** — использует текущее активное подключение ST без отдельного ручного профиля
- собственный профиль STMB — позволяет, например, использовать более дешёвую модель для памяти, чем для основного ролевого чата
- скрытие/показ сообщений — помогает экономить токены и держать чат чище
- **`Manual Lorebook Mode`** и **`Auto-create lorebook if none exists`** — определяют, куда будут сохраняться памяти
- **`Показывать предпросмотр памяти`** — даёт просмотреть и поправить результат перед сохранением
- **`Автоматически создавать сводки памяти`** — включает автоматическое создание памяти
- **`Интервал автосводки`** и **`Буфер автосводки`** — управляют моментом автогенерации
- **`Боковые промпты`** — включают трекеры и служебные записи

## 🔧 Решение проблем (когда что-то не работает)

Этот гайд **не является полной матрицей диагностики**. Подробный список есть в [readme-RU.md](readme-RU.md).

Самые быстрые проверки:

- убедитесь, что STMB включён и пункт **Memory Books** виден в меню расширений
- если авто-сводки не запускаются, проверьте, что вы сначала создали хотя бы одну память вручную и что интервал/буфер выставлены разумно
- если память не сохраняется, убедитесь, что у чата привязан lorebook или включён **`Auto-create lorebook if none exists`**
- если записи не триггерятся, проверьте, что `Delay until recursion` не мешает активации
- если Regex ведёт себя странно, проверьте выбор скриптов в **`📐 Настроить regex…`**, а не только настройки Regex extension
- если не появляется prompt на консолидацию, проверьте **`Запрашивать консолидацию, когда уровень готов`** и выбранные уровни в **`Уровни автоматической консолидации`**
