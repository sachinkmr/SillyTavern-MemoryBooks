# 📕 ST Memory Books - 您的 AI 聊天記憶助手

**將漫長的聊天對話整理成有條理、可搜尋的記憶！**

需要讓機器人記住事情，但聊天內容太長，已經超出上下文能負荷的範圍？想要自動追蹤重要劇情點，又不想自己手動做筆記？ST Memory Books 就是為此而設計的。它會觀察您的聊天，建立智慧摘要，讓您不再輕易忘記故事中發生過的事。

## 🚀 快速開始（5 分鐘建立您的第一筆記憶！）

**第一次使用 ST Memory Books？** 只要幾個步驟，就可以先把自動記憶跑起來。

### 第一步：找到擴充功能

- 在聊天輸入框旁邊找到魔杖圖示（🪄）
- 點開它，然後點選 **Memory Books**
- 您會看到 ST Memory Books 的控制面板

### 第二步：開啟自動模式

- 在控制面板中找到 **自動建立記憶摘要**
- 把它打開
- 把 **自動摘要間隔** 設成 **20-30 則訊息**（這是很好的起始值）
- 一開始先把 **自動摘要緩衝區** 維持在低值（`0-2` 很適合新手）
- 先手動建立一筆記憶，讓這個聊天完成初始化
- 這樣就可以了！🎉

### 第三步：像平常一樣聊天

- 繼續正常聊天
- 當新增訊息累積到大約 20-30 則後，ST Memory Books 會自動：
  - 使用自上次處理完成之後的新訊息
  - 請您的 AI 寫出摘要
  - 把摘要存進記憶集合
  - 在完成時顯示通知

**恭喜！** 您現在已經有自動記憶管理了。之後就比較不容易忘記幾個章節之前發生過什麼。

## 💡 ST Memory Books 實際上做了什麼

您可以把 ST Memory Books 想成聊天對話的 **AI 私人圖書管理員**。

### 🤖 **自動摘要**
*「我不想一直管它，只想讓它自己運作」*

- 在背景監控聊天
- 每隔一段訊息數自動建立記憶
- 適合長篇角色扮演、創作寫作、持續進行的故事

### ✋ **手動建立記憶**
*「我想自己決定哪些內容要被保存」*

- 用簡單的箭頭按鈕（► ◄）標記重要場景
- 在重要時刻手動建立記憶
- 很適合保存關鍵劇情點或角色成長片段

### 📊 **側邊提示詞與智慧追蹤器**
*「我想追蹤關係、劇情線、狀態或數值」*

- 可重複使用的提示片段，幫助記憶生成
- 內建範本庫，可直接拿來用
- 可以自訂 AI 提示，追蹤任何您想追蹤的內容
- 會自動更新記分板、關係狀態、劇情摘要等資訊
- 例如：「誰喜歡誰？」「目前任務狀態」「角色心情追蹤」

### 📚 **記憶集合**
*所有記憶存放的地方*

- 會自動整理，方便搜尋
- 與 SillyTavern 內建 lorebook 系統整合
- 您的 AI 可以在後續聊天中參考過往記憶

**您會得到：**

- 不容易漏掉重要劇情
- 單人聊天與群組聊天都能用

**小建議：** 可以先從 30 則訊息開始，再依照聊天速度調整。聊天節奏很快時可以往上調，聊天節奏很慢、內容很細時可以往下調。

**常用指令：**

**您會得到：**
- 更快速的記憶建立流程
- 比較容易批次處理
- 更方便接進自己的工作流

## 🙈 節省 Token：隱藏 / 取消隱藏訊息

在長聊天中，最簡單的節省 token 方法之一，就是把**已經做成記憶的舊訊息隱藏起來**。

### 「隱藏」是什麼意思？

隱藏訊息 **不是刪除**。原始聊天仍然存在，記憶也仍然在 lorebook 裡。只是那些原始訊息不再直接送給 AI。

### 什麼時候會有幫助？

隱藏 / 取消隱藏功能很適合以下情況：

- 聊天已經變得很長
- 那些訊息已經做成記憶
- 您想讓聊天畫面更乾淨

### 建立記憶後自動隱藏

STMB 可以在建立記憶後自動隱藏訊息。您可以選擇：

- **不自動隱藏**：全部保持可見（您也可以自己用 `/hide x-y` 手動隱藏）
- **自動隱藏直到最後一筆記憶為止的所有訊息**：隱藏所有已被記憶覆蓋的內容
- **僅自動隱藏最後一筆記憶中的訊息**：只隱藏最近剛處理完的範圍

您也可以用 **保留可見訊息數** 來決定有多少最近訊息要保持顯示。

### 建立記憶前先取消隱藏

**建立記憶前取消隱藏訊息** 這個設定會在產生記憶前，暫時對選取範圍執行 `/unhide X-Y`。  
如果您常常重做記憶，或聊天中有不少已隱藏內容，這個設定會很有用。

### 新手推薦設定

Aiko 自己偏好的做法是：

- 使用 **自動隱藏直到最後一筆記憶為止的所有訊息**
- 保留 **2 則訊息** 可見
- 開啟 **建立記憶前取消隱藏訊息**

## 🌈 總結整合

總結整合可以把較舊的 STMB 記憶壓縮成更高層級的回顧摘要，讓長篇故事更容易管理。

### 問：什麼是總結整合？

**答：** STMB 不只是一直做場景等級的記憶。它也可以把既有記憶或既有總結，再往上整合成更精簡的回顧。第一層是 **Arc**，更長篇故事還可以往上做成：

- Arc
- Chapter
- Book
- Legend
- Series
- Epic

### 問：為什麼要用？

**答：** 總結整合在以下情況特別有用：

- 您的記憶列表已經很長
- 舊條目已經不需要逐場景的細節
- 您想降低 token 使用量，但又不想失去故事連貫性
- 您想要更高層級、更乾淨的劇情回顧

### 問：它會自動執行嗎？

**答：** 不會。總結整合仍然需要您的確認。

- 您可以隨時從主視窗手動打開 **整合記憶**
- 您也可以啟用 **當某個層級準備好時提示進行整合**
- 當某個已選擇的目標層級，達到其已儲存的最小來源數量時，STMB 會顯示 **yes/later** 確認提示
- 按下 **Yes** 只會幫您打開整合視窗，並預先選好該層級；它不會靜默自動跑完

### 問：我要怎麼使用？

**答：** 建立一筆整合摘要時：

1. 在 STMB 主視窗中點擊 **整合記憶**
2. 選擇目標摘要層級
3. 挑選您要納入的來源條目
4. 如有需要，可在新摘要建立後停用來源條目
5. 點擊 **Run**

如果 AI 回傳了不理想的整合結果，STMB 也可以先讓您檢查 / 修正，再重新提交。

### 什麼會被整合？什麼不會？

總結整合處理的是 **STMB 記憶** 與 **STMB 摘要**。

也就是說：

- 一般記憶可以被整合成更高層級的摘要
- 更高層級的摘要之後也可以再往上整合成更大的回顧

Side Prompt 不一樣。

**Side Prompt 是追蹤器條目**，不是記憶摘要條目。它們的用途是持續維護某些狀態資訊，例如：

- 關係狀態
- 目前目標
- 記分板
- 世界狀態
- 劇情線追蹤

用最直白的方式說：

- **記憶** =「這個場景發生了什麼？」
- **Side Prompt** =「這個東西現在是什麼狀態？」
- **總結整合** =「把很多記憶濃縮成更大的回顧摘要」

### 為什麼這點很重要？

因為有些使用者會以為追蹤器條目也會一起被捲進 Arc 或 Chapter 總結裡。

它們 **不會**。

如果您想讓某個內容參與整合，它必須是一般的 STMB 記憶 / 摘要條目，而不只是 Side Prompt。

### 簡單判斷原則

請這樣分：

- **記憶**：用來保存場景回顧
- **Side Prompt**：用來維護持續更新的追蹤資訊
- **總結整合**：用來把舊記憶壓縮成更高層級的回顧

## 🎨 追蹤器、側邊提示詞與範本（進階功能）

**側邊提示詞** 是背景追蹤器，用來維持故事中持續變動的資訊。  
它們會在記憶建立流程旁邊一起運作，並逐步更新獨立的側邊提示詞故事書條目。您可以把它想成是**一直看著故事、負責幫您記住某些狀態的助手**。

### 🚀 **用範本快速開始**

1. 打開 Memory Books 設定
2. 點擊 **側邊提示詞**
3. 瀏覽 **範本庫**，挑選適合您故事的內容：

   - **Character Development Tracker** - 追蹤角色個性與成長變化
   - **Relationship Dynamics** - 追蹤角色之間的關係
   - **Plot Thread Tracker** - 追蹤持續進行中的劇情線
   - **Mood & Atmosphere** - 追蹤情緒與氛圍
   - **World Building Notes** - 追蹤世界觀與設定細節
4. 啟用您要使用的範本（之後隨時可以再修改）
5. 如果範本使用自動觸發，STMB 就會在記憶建立過程中一起更新該 side-prompt 條目

[Scribe：逐步示範如何啟用自動側邊提示詞](https://scribehow.com/viewer/How_to_Enable_Side_Prompts_in_Memory_Books__fif494uSSjCmxE2ZCmRGxQ)

### ⚙️ **側邊提示詞怎麼運作**

- **背景追蹤器**：它們會安靜地在背景更新資訊
- **不干擾主體**：不會直接改動您的主要 AI 設定或角色提示
- **可依聊天分開控制**：不同聊天可以啟用不同追蹤器
- **以範本為基礎**：可以使用內建範本，也可以自己建立
- **可自動也可手動**：標準範本可以自動執行；帶有自訂 runtime 巨集的範本則只能手動執行
- **巨集支援**：`Prompt`、`Response Format`、`Title` 與關鍵字欄位都可以使用標準 ST 巨集，例如 `{{user}}`、`{{char}}`
- **Runtime 巨集**：非標準 `{{...}}` 會變成執行時必填輸入，例如 `{{npc name}}="Jane Doe"`
- **可回傳純文字**：側邊提示詞不一定要回傳 JSON
- **覆寫式更新**：側邊提示詞會持續更新自己的追蹤條目，而不是每次都新增一筆新的連續記憶

### 🛠️ **管理側邊提示詞**

- **側邊提示詞管理器**：建立、編輯、複製、整理追蹤器
- **啟用 / 停用**：可隨時打開或關閉追蹤器
- **匯入 / 匯出**：方便分享或備份範本
- **狀態檢視**：查看目前聊天中哪些追蹤器已啟用、何時運作
- **安全檢查**：如果範本包含自訂 runtime 巨集，STMB 會在儲存 / 匯入時移除其自動觸發設定，並顯示警告 toast

### 💡 **範本示例**

- Side Prompt 範本庫（可直接匯入這份 JSON）：
  [SidePromptTemplateLibrary.json](../resources/SidePromptTemplateLibrary.json)

一些常見提示思路：

- 「追蹤重要對話與角色互動」
- 「持續更新目前任務狀態」
- 「記下新出現的世界觀細節」
- 「追蹤角色 A 與角色 B 的關係」

### 🔧 **建立自訂側邊提示詞**

1. 打開側邊提示詞管理器
2. 點擊 **新建**
3. 寫下簡短且清楚的指令  
   （例如：「每個場景都記下天氣狀態」）
4. 如有需要，可加入標準 ST 巨集，例如 `{{user}}` 或 `{{char}}`
5. 如果您加入自訂 runtime 巨集，例如 `{{location name}}`，就必須用 `/sideprompt "Name" {{location name}}="value"` 手動執行
6. 儲存並啟用
7. 如果它使用的是自動觸發，就會在之後持續更新；否則就要在需要時手動執行

### 💬 **小建議**

側邊提示詞 最適合拿來做**小而專注**的事情。  
與其寫成「追蹤所有東西」，通常更好的做法是像「追蹤主角之間的曖昧張力」這樣具體。

### ⌨️ **手動 `/sideprompt` 語法**

使用：
`/sideprompt "Name" {{macro}}="value" [X-Y]`

範例：

- `/sideprompt "Status" 10-20`
- `/sideprompt "NPC Directory" {{npc name}}="Jane Doe" 40-50`
- `/sideprompt "Location Notes" {{place name}}="Black Harbor" 100-120`

注意：

- 側邊提示詞名稱必須加引號
- Runtime 巨集的值也必須加引號
- 選好側邊提示詞之後，slash 指令自動完成會提示您補上需要的 runtime 巨集
- 如果範本包含自訂 runtime 巨集，STMB 會強制讓它只能手動執行，並移除自動觸發
- `X-Y` 是可選的；如果省略，STMB 會使用該 Side Prompt 上次更新之後的訊息

---

### 🧠 使用 Regex 擴充功能做進階文字控制

**想更精細地控制 STMB 傳給 AI 的文字，以及 AI 回傳後 STMB 如何處理它？**  
STMB 可以在生成前與儲存前，執行您挑選的 Regex 腳本。

這對以下情況很有用：

- 清除 AI 回應裡重複、沒用的雜訊
- 在生成前統一名稱或術語
- 在 STMB 預覽 / 解析前先整理文字格式

#### **現在的用法**

1. 先在 SillyTavern 的 **Regex** 擴充功能裡建立您要的腳本
2. 在 STMB 中打開 **使用正則表達式（進階）**
3. 點擊 **📐 設定正則表達式…**
4. 選擇 STMB 要執行哪些腳本：
   - 在送給 AI 之前執行
   - 在加入 lorebook 之前執行

#### **重要行為**

- STMB 的 Regex 選擇是由 **STMB 自己管理**
- 某個腳本即使在 Regex 擴充功能中被停用，只要它在 STMB 內被選中，仍然可能執行
- STMB 的送出前 / 儲存前兩側都支援多選

#### **快速示例**

如果您的模型總是加上一句 `(OOC: I hope this summary is helpful!)`，您可以：

1. 建立一個會移除這段文字的 Regex 腳本
2. 在 STMB 中打開 **使用正則表達式（進階）**
3. 點開 **📐 設定正則表達式…**
4. 把這個腳本加進 **incoming** 選單

之後 STMB 就會在預覽或儲存前，先把這段文字清掉。

## ⚙️ 先學這些重要設定

這份使用指南不是完整設定手冊。  
如果您想看逐項設定說明，請改看 [繁體中文 README](readme-zh-tw.md)。

大多數使用者最值得先理解的設定是：

- **自動建立記憶摘要**：決定是否開啟自動建立記憶
- **自動摘要間隔** 與 **自動摘要緩衝區**：控制自動建立發生的時機
- **顯示記憶預覽**：讓您在儲存前先檢查或修改 AI 輸出
- **當某個層級準備好時提示進行整合** 與 **自動整合層級**：在整合條件成熟時提醒您，但不會靜默執行
- **啟用手動 Lorebook 模式** 與 **若無 lorebook 則自動建立**：決定記憶要存去哪裡
- **使用正則表達式（進階）**：打開由 STMB 管理的 Regex 選擇
- **目前 SillyTavern 設定**：直接使用您目前正在使用的 ST 連線設定，而不另外建立自訂提供者設定檔

## 🔧 疑難排解（當事情沒有正常運作時）

這份使用指南不是完整疑難排解清單。  
詳細版本請看 [繁體中文 README](readme-zh-tw.md)。

最快的基本檢查方式如下：

- 確認 STMB 已啟用，而且 **Memory Books** 出現在擴充功能選單中
- 如果自動摘要沒有觸發，先確認您已手動建立過一筆記憶，並檢查 interval / buffer 是否合理
- 如果記憶無法儲存，確認目前聊天有綁定 lorebook，或已啟用 **若無 lorebook 則自動建立**
- 如果 Regex 行為看起來不對，請檢查 **📐 設定正則表達式…** 裡的選項，而不只是看 Regex 擴充功能本身
- 如果整合提示沒有出現，請確認 **當某個層級準備好時提示進行整合** 已打開，而且目標層級有包含在 **自動整合層級** 裡
